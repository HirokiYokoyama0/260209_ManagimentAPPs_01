## 🎯 読む順番（推奨）

### 新規参加者の場合
1. **00_README.md**（このファイル）- ドキュメント案内
2. **00_ファイル構成.md** - プロジェクト全体の構造把握
3. **02_クイックスタート.md** - 環境構築・起動
4. **04_Supabaseセットアップ手順.md** - DB設定
5. **03_管理ダッシュボード仕様書.md** - 機能理解

### 機能追加・改善を行う場合
1. **03_管理ダッシュボード仕様書.md** - 現在の仕様確認
2. **99_変更履歴.md** - 最新の実装状況確認
3. **00_ファイル構成.md** - 必要に応じてコード位置確認
4. 実装後、**99_変更履歴.md**を更新

## 🗄️ アーカイブについて

`archive/` フォルダには、統合・整理前のオリジナルファイルが保管されています。
通常の開発では参照不要ですが、過去の経緯を確認したい場合にご利用ください。

## ✏️ ドキュメント更新ルール

- 機能追加・変更時は、必ず該当する仕様書を更新してください
- 重要な変更は **99_変更履歴.md** に記録してください
- ドキュメントの日付形式は `YYYY-MM-DD` で統一してください
