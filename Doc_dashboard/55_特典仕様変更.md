## 📢 変更概要

特典システムを**「手動交換型」**から**「マイルストーン自動付与型」**に変更します。

### 変更前（現行仕様）

**「手動交換型」**
- ユーザーが好きなタイミングで特典を選んで「交換申請」
- 固定の特典メニュー（5, 10, 15, 20 スタンプ）
- スタンプ数は減らない（積み上げ式）
- 交換履歴を `reward_exchanges` テーブルに記録
- 既存の管理画面・APIは維持可能

| 特典名 | 必要スタンプ数 |
|--------|--------------|
| オリジナル歯ブラシセット | 5 |
| フッ素塗布1回無料券 | 10 |
| 歯のクリーニング50%OFF券 | 15 |
| ホワイトニング1回30%OFF券 | 20 |

### 変更後（新仕様）

**「マイルストーン自動付与型」**
- **累積スタンプ数**が一定の節目（10, 50, 300...）に到達したら**システムが自動的に特典を付与**
- ユーザーは「交換申請」ではなく「受け取る」のみ
- スタンプ数は引き続き減らない（積み上げ式）
- 同じ `reward_exchanges` テーブルに自動付与の履歴を記録（**既存コード流用可能**）
- 既存の「完了」「キャンセル」処理はそのまま使用

## 🎯 新しい特典ルール

### 特典付与タイミング

| 累積スタンプ数 | 特典内容 | 詳細・条件 | 有効期限 |
|--------------|---------|----------|----------|
| **10, 20, 30, 40...** | 歯ブラシ 1本 | TePeミニ / コンパクト / ピセラ / 替えブラシ 等から選択 | **当日限り** |
| **50, 100, 150, 200...** | POIC殺菌剤 | 初回のみ本体込、2回目以降は補充用のみ | **5ヶ月間** |
| **300** | 選べる自費メニュー | 下記3点から1つ選択 | **5ヶ月間** |
| **450, 600, 750...** | 選べる自費メニュー | 300以降、150の倍数ごとに継続付与 | **5ヶ月間** |

**⚠️ 重要な付与ルール:**
- 複数のマイルストーンが重なる場合（例: 50は10の倍数でもあり50の倍数でもある）、**優先度が高い特典を1つのみ**付与します
- **優先度:** 選べる自費メニュー > POIC殺菌剤 > 歯ブラシ

### 選べる自費メニュー詳細

300ポイント到達時および以降150の倍数ごとに、以下から1つ選択可能：

1. **小顔エステ 10%OFF**
2. **ホワイトニング 10%OFF**
3. **その他自費 10%OFFクーポン**

**⚠️ 重要な有効期限について:**
- **歯ブラシ:** 当日限り有効（受け取った日のみ使用可能）
- **POIC殺菌剤:** 付与日から5ヶ月間有効
- **自費メニュー:** 付与日から5ヶ月間有効
- アプリ画面上に明確な期限を表示し、期限切れ防止のため利用を促すUI/UX設計が必要

### 例2: 優先度ルールの詳細説明

**50スタンプ到達時:**
- 該当マイルストーン: 10の倍数（歯ブラシ）、50の倍数（POIC）
- **付与される特典:** POIC殺菌剤（本体込）のみ
- **付与されない:** 歯ブラシ（優先度が低いため）

**100スタンプ到達時:**
- 該当マイルストーン: 10の倍数（歯ブラシ）、50の倍数（POIC）
- **付与される特典:** POIC殺菌剤（補充用）のみ
- **付与されない:** 歯ブラシ（優先度が低いため）

**300スタンプ到達時:**
- 該当マイルストーン: 10の倍数（歯ブラシ）、50の倍数（POIC）、300（自費メニュー）
- **付与される特典:** 選べる自費メニューのみ
- **付与されない:** 歯ブラシ、POIC（優先度が低いため）

**⚠️ 重要:**
- 特典は到達時に**1つのみ**付与されます
- 優先度が高い特典のみが付与され、低い特典はスキップされます
- スキップされた特典は次のマイルストーンでも獲得できません

#### 1. `rewards` テーブル（特典マスター）の見直し

**現行の構造:**
```sql
CREATE TABLE rewards (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT,
  required_stamps INTEGER NOT NULL,  -- 固定の必要スタンプ数
  stock_count INTEGER,               -- 在庫数
  is_active BOOLEAN DEFAULT true,
  image_url TEXT,
  display_order INTEGER,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**新仕様での変更案（Option A: 既存テーブル拡張）**

```sql
ALTER TABLE rewards
  -- マイルストーン型の特典かどうか
  ADD COLUMN reward_type TEXT CHECK (reward_type IN ('fixed', 'milestone')) DEFAULT 'fixed',

  -- マイルストーンルール（10の倍数、50の倍数、など）
  ADD COLUMN milestone_rule TEXT, -- 'every_10', 'every_50', 'initial_300_then_150'

  -- 初回のみ特別扱いか（例: POIC初回は本体込）
  ADD COLUMN is_first_time_special BOOLEAN DEFAULT false,
  ADD COLUMN first_time_description TEXT,

  -- 有効期限（月数）
  ADD COLUMN validity_months INTEGER, -- NULL = 無期限

  -- 在庫管理を無効化（マイルストーン型の場合）
  -- stock_count は milestone 型では使用しない
;
```

**新仕様での変更案（Option B: 新規テーブル作成）**

既存の `rewards` は削除せず、新しく `milestone_rewards` テーブルを作成：

```sql
CREATE TABLE milestone_rewards (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  description TEXT,

  -- マイルストーンルール
  milestone_rule TEXT NOT NULL, -- 'every_10', 'every_50', 'initial_300_then_150'

  -- 初回特別対応
  is_first_time_special BOOLEAN DEFAULT false,
  first_time_description TEXT,
  subsequent_description TEXT,

  -- 有効期限
  validity_months INTEGER, -- NULL = 無期限

  -- 表示順・有効フラグ
  display_order INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT true,

  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 2. `reward_exchanges` テーブルの見直し

**現行:** ユーザーの「交換申請」を記録
**新仕様:** ユーザーへの「特典付与」を記録

**変更案（カラム追加）:**

```sql
ALTER TABLE reward_exchanges
  -- どのマイルストーンで付与されたか
  ADD COLUMN milestone_reached INTEGER, -- 10, 50, 300, 450...

  -- 受け取り済みかどうか（pending → claimed → used）
  -- status の値を拡張: 'pending' → 'claimed' → 'used'

  -- 有効期限
  ADD COLUMN valid_until TIMESTAMPTZ, -- NULL = 無期限

  -- 初回特典かどうか（POIC本体込など）
  ADD COLUMN is_first_time BOOLEAN DEFAULT false
;

-- status の CHECK 制約を更新
ALTER TABLE reward_exchanges
DROP CONSTRAINT IF EXISTS reward_exchanges_status_check;

ALTER TABLE reward_exchanges
ADD CONSTRAINT reward_exchanges_status_check
CHECK (status IN ('pending', 'claimed', 'used', 'expired', 'cancelled'));
```

**ステータスの意味:**

| status | 意味 | 説明 |
|--------|-----|------|
| `pending` | 特典解放済み（未受け取り） | マイルストーン到達で自動付与、アプリに通知表示 |
| `claimed` | 受け取り済み（未使用） | ユーザーがアプリで「受け取る」ボタン押下 |
| `used` | 使用済み | 受付で特典を引き渡し完了 |
| `expired` | 期限切れ | 有効期限切れ |
| `cancelled` | キャンセル | 在庫切れなど |

#### 3. 新規テーブル: `milestone_history` （到達履歴）

各ユーザーがどのマイルストーンに到達したかを記録：

```sql
CREATE TABLE milestone_history (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id TEXT NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  milestone INTEGER NOT NULL, -- 10, 20, 50, 100, 300...
  reached_at TIMESTAMPTZ DEFAULT NOW(),

  -- このマイルストーンで付与された特典のID（複数ある場合は配列）
  reward_exchange_ids UUID[] DEFAULT '{}',

  created_at TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(user_id, milestone) -- 同じマイルストーンは1回のみ
);

CREATE INDEX idx_milestone_history_user ON milestone_history(user_id);
CREATE INDEX idx_milestone_history_milestone ON milestone_history(milestone);
```

## 🔧 実装の流れ

### Step 1: マイルストーン判定ロジック（優先度対応）

スタンプ数が変更されたタイミングで、到達したマイルストーンを判定し、優先度に基づいて1つの特典のみを選択：

```typescript
// 特典の種類
enum RewardType {
  TOOTHBRUSH = 'toothbrush',    // 歯ブラシ
  POIC = 'poic',                 // POIC殺菌剤
  PREMIUM_MENU = 'premium_menu'  // 選べる自費メニュー
}

// 優先度: 数値が大きいほど優先度が高い
const REWARD_PRIORITY = {
  [RewardType.TOOTHBRUSH]: 1,
  [RewardType.POIC]: 2,
  [RewardType.PREMIUM_MENU]: 3
};

// マイルストーンごとの特典タイプを判定
function getRewardTypeForMilestone(milestone: number): RewardType {
  // 300以降150の倍数チェック（最優先）
  if (milestone === 300 || (milestone > 300 && (milestone - 300) % 150 === 0)) {
    return RewardType.PREMIUM_MENU;
  }

  // 50の倍数チェック（次優先）
  if (milestone % 50 === 0) {
    return RewardType.POIC;
  }

  // 10の倍数チェック（最低優先）
  if (milestone % 10 === 0) {
    return RewardType.TOOTHBRUSH;
  }

  throw new Error(`Invalid milestone: ${milestone}`);
}

// スタンプ数変更時にマイルストーンを判定し、優先度に基づいて特典を選択
function checkMilestones(oldStampCount: number, newStampCount: number): Array<{
  milestone: number;
  rewardType: RewardType;
}> {
  const results: Array<{ milestone: number; rewardType: RewardType }> = [];

  // 通過した全マイルストーンを取得
  const allMilestones: number[] = [];

  // 10の倍数を収集
  const old10 = Math.floor(oldStampCount / 10);
  const new10 = Math.floor(newStampCount / 10);
  for (let i = old10 + 1; i <= new10; i++) {
    allMilestones.push(i * 10);
  }

  // 重複を除去してソート
  const uniqueMilestones = [...new Set(allMilestones)].sort((a, b) => a - b);

  // 各マイルストーンに対して優先度の高い特典を選択
  for (const milestone of uniqueMilestones) {
    const rewardType = getRewardTypeForMilestone(milestone);
    results.push({ milestone, rewardType });
  }

  return results;
}

// 使用例1: 95 → 105 に変更
const reached1 = checkMilestones(95, 105);
// => [{ milestone: 100, rewardType: 'poic' }]
// 100は10の倍数でもあり50の倍数でもあるが、優先度の高いPOICを選択

// 使用例2: 45 → 55 に変更
const reached2 = checkMilestones(45, 55);
// => [{ milestone: 50, rewardType: 'poic' }]
// 50は10の倍数でもあるが、POICを優先

// 使用例3: 10 → 25 に変更
const reached3 = checkMilestones(10, 25);
// => [{ milestone: 20, rewardType: 'toothbrush' }]
// 20は10の倍数のみなので歯ブラシ

// 使用例4: 290 → 310 に変更
const reached4 = checkMilestones(290, 310);
// => [
//   { milestone: 300, rewardType: 'premium_menu' }
// ]
// 300は10の倍数、50の倍数、300でもあるが、最優先の自費メニューを選択
// 310の歯ブラシも通過するが、別のマイルストーンとして付与される
```

### Step 2: 特典の自動付与（優先度ルール適用）

マイルストーン到達時に、優先度に基づいて**1つの特典のみ**を `reward_exchanges` に作成：

```typescript
async function grantMilestoneReward(
  userId: string,
  milestone: number,
  rewardType: RewardType
) {
  // 特典タイプに応じてマスターデータを取得
  const reward = await getRewardByType(rewardType);

  if (!reward) {
    throw new Error(`Reward not found for type: ${rewardType}`);
  }

  // 初回判定（POIC用）
  let isFirstTime = false;
  if (rewardType === RewardType.POIC) {
    isFirstTime = await checkIfFirstTimePOIC(userId);
  }

  // 有効期限計算
  let validUntil = null;
  if (rewardType === RewardType.TOOTHBRUSH) {
    // 歯ブラシ: 当日限り（翌日0時まで）
    validUntil = new Date();
    validUntil.setHours(23, 59, 59, 999);
  } else if (rewardType === RewardType.POIC || rewardType === RewardType.PREMIUM_MENU) {
    // POIC・自費メニュー: 5ヶ月間
    validUntil = addMonths(new Date(), 5);
  }

  // 特典付与レコード作成
  const { data, error } = await supabase.from('reward_exchanges').insert({
    user_id: userId,
    reward_id: reward.id,
    milestone_reached: milestone,
    status: 'pending',
    valid_until: validUntil,
    is_first_time: isFirstTime,
    notes: `${milestone}スタンプ到達で自動付与（${rewardType}）`,
    exchanged_at: new Date().toISOString()
  });

  if (error) {
    throw new Error(`Failed to grant reward: ${error.message}`);
  }

  // マイルストーン履歴に記録
  await supabase.from('milestone_history').insert({
    user_id: userId,
    milestone: milestone,
    reward_exchange_ids: data ? [data.id] : [],
    reached_at: new Date().toISOString()
  });

  return data;
}

// スタンプ付与時の処理フロー全体
async function onStampCountChanged(
  userId: string,
  oldCount: number,
  newCount: number
) {
  // マイルストーンを判定
  const milestones = checkMilestones(oldCount, newCount);

  // 各マイルストーンに対して特典を付与
  for (const { milestone, rewardType } of milestones) {
    await grantMilestoneReward(userId, milestone, rewardType);
  }

  // ユーザーに通知（LINE Push APIなど）
  if (milestones.length > 0) {
    await sendMilestoneNotification(userId, milestones);
  }
}

// POIC初回判定
async function checkIfFirstTimePOIC(userId: string): Promise<boolean> {
  const { data } = await supabase
    .from('reward_exchanges')
    .select('id')
    .eq('user_id', userId)
    .eq('reward_id', POIC_REWARD_ID) // POIC特典のID
    .limit(1);

  return !data || data.length === 0;
}

// 特典タイプから特典マスターを取得
async function getRewardByType(rewardType: RewardType) {
  const rewardNameMap = {
    [RewardType.TOOTHBRUSH]: '歯ブラシ 1本',
    [RewardType.POIC]: 'POIC殺菌剤',
    [RewardType.PREMIUM_MENU]: '選べる自費メニュー'
  };

  const { data } = await supabase
    .from('milestone_rewards')
    .select('*')
    .eq('name', rewardNameMap[rewardType])
    .single();

  return data;
}
```

## ⚠️ 注意点・検討事項

### 1. 既存ユーザーデータの扱い

**問題:**
- 現在のユーザーは累積スタンプ数に応じて、過去のマイルストーンをすべて達成している状態

**対応案:**

#### Option A: 既存ユーザーは現在のスタンプ数以降のみ対象

```
例: 現在150スタンプ持っているユーザー
→ 0〜150のマイルストーン特典は付与しない
→ 次は200スタンプで特典付与開始
```

**メリット:** 実装が簡単
**デメリット:** 既存ユーザーが不公平感を持つ可能性

#### Option B: 既存ユーザーにも過去分を遡及付与

```
例: 現在150スタンプ持っているユーザー
→ 10, 20, 30...150 までのすべての特典を一度に付与
```

**メリット:** 公平
**デメリット:** 特典が大量に付与される、在庫問題

#### Option C: 既存ユーザーには移行ボーナス

```
例: 現在150スタンプ持っているユーザー
→ 特別な「移行記念特典」を1回のみ付与
→ 次は160スタンプで通常の特典付与開始
```

**メリット:** 公平感を保ちつつ在庫問題を回避
**推奨:** Option C

### 2. POIC初回判定ロジック

**問題:**
- 50スタンプ到達時は本体込、100スタンプ以降は補充用のみ

**実装:**
- `reward_exchanges` から該当ユーザーの POIC 受け取り履歴を確認
- 初回なら `is_first_time=true` で「本体込」
- 2回目以降は `is_first_time=false` で「補充用」

### 3. 300 + 150の倍数ルール

**実装方法:**
```typescript
function isHighTierMilestone(stampCount: number): boolean {
  if (stampCount === 300) return true;
  if (stampCount > 300 && (stampCount - 300) % 150 === 0) return true;
  return false;
}
```

### 4. 有効期限の管理

**自動期限切れ処理:**

```sql
-- 定期実行（日次バッチ）
UPDATE reward_exchanges
SET status = 'expired'
WHERE status IN ('pending', 'claimed')
  AND valid_until IS NOT NULL
  AND valid_until < NOW();
```

**期限切れ前の通知（1週間前など）:**
- LINE プッシュ通知
- アプリ内アラート表示
