# 仕様変更: 管理ダッシュボードでもマイルストーン特典を自動付与

**作成日**: 2026-04-04
**変更理由**: ユーザー要望による仕様変更
**影響範囲**: 管理ダッシュボードのスタンプ操作API

---

## 📋 変更概要

### 変更前（112-114番ドキュメントの仕様）
- **管理ダッシュボード**: マイルストーン特典を**自動付与しない**
- **LIFFアプリ**: マイルストーン特典を自動付与する
- **理由**: スタッフ操作とユーザー操作を区別するため

### 変更後（本ドキュメント）
- **管理ダッシュボード**: マイルストーン特典を**自動付与する** ✅
- **LIFFアプリ**: マイルストーン特典を自動付与する
- **理由**: ユーザーの運用ニーズに合わせて統一

---

### 修正ファイル

#### 1. `app/api/profiles/[id]/stamp/route.ts`
スタンプ増減API（+1/-1ボタン）

**変更内容**:

```typescript
// Before
// ⚠️ スタッフ操作では自動付与しない（LIFFアプリ側の仕様に統一）
if (newCount < currentStampCount) {
  await invalidateMilestoneRewards(id, currentStampCount, newCount);
}

// After
// 6️⃣ マイルストーン特典の処理
if (newCount < currentStampCount) {
  // スタンプ減少時：該当する特典を削除
  await invalidateMilestoneRewards(id, currentStampCount, newCount);
} else if (newCount > currentStampCount) {
  // スタンプ増加時：マイルストーン到達を判定して特典を自動付与
  const milestones = await handleStampMilestones(id, currentStampCount, newCount);
  if (milestones.length > 0) {
    console.log(`✅ マイルストーン特典を ${milestones.length}件 付与しました`);
  }
}
```

## 📊 動作フロー

### スタンプ増加時（例: 8個 → 12個）

1. **スタンプ履歴に記録**
   - `stamp_history` テーブルに INSERT

2. **プロフィール更新**
   - `profiles.stamp_count` を 12 に更新

3. **マイルストーン判定** ✅ NEW
   - 8個 < **10個** ≤ 12個 → **10個マイルストーン到達**
   - `handleStampMilestones()` を呼び出し

4. **特典自動付与** ✅ NEW
   - `reward_exchanges` テーブルに INSERT
   - `status: 'available'` で作成
   - `milestone_reached: 10`
   - `is_milestone_based: true`

### スタンプ減少時（例: 25個 → 5個）

1. **スタンプ履歴削除**
   - `stamp_history` の stamp_number > 5 のレコードを DELETE

2. **プロフィール更新**
   - `profiles.stamp_count` を 5 に更新

3. **マイルストーン特典削除**
   - `invalidateMilestoneRewards()` を呼び出し
   - milestone_reached > 5 の特典（10個、20個）を DELETE
   - 対象: `status = 'available'` または `'pending'`

---

## ⚙️ 自動付与される特典のステータス

```typescript
status: 'available'  // ✅ 正しい初期ステータス
```

- **available**: 「この特典と交換する」ボタンが表示される状態
- **pending**: ユーザーが「交換申請」した後の状態
- **completed**: スタッフが引き渡し完了した状態

## 🔍 具体例

### ケース1: スタッフが0個 → 25個に変更

**操作**:
```
管理ダッシュボード > 患者一覧 > スタンプ数を「25」に設定
```

**結果**:
```sql
-- reward_exchanges に自動挿入される
INSERT INTO reward_exchanges (
  user_id,
  reward_id,
  milestone_reached,
  status,
  is_milestone_based,
  ...
) VALUES
  (..., '10個の特典ID', 10, 'available', true, ...),
  (..., '20個の特典ID', 20, 'available', true, ...);
```

**画面表示**（LIFFアプリ）:
- 10個マイルストーン: 「この特典と交換する」ボタン表示
- 20個マイルストーン: 「この特典と交換する」ボタン表示

### ケース2: スタッフが25個 → 0個に変更

**操作**:
```
管理ダッシュボード > 患者一覧 > スタンプ数を「0」に設定
```

**結果**:
```sql
-- reward_exchanges から削除される
DELETE FROM reward_exchanges
WHERE user_id = '...'
  AND is_milestone_based = true
  AND milestone_reached > 0
  AND status IN ('available', 'pending');
```

**画面表示**（LIFFアプリ）:
- すべての特典が消える（引渡済を除く）

## ⚠️ 注意事項

### 削除されない特典

以下の特典は、スタンプを減らしても**削除されません**:

1. **引渡済（completed）**
   - すでにスタッフが引き渡した特典
   - 履歴として残すべきデータ

2. **キャンセル済（cancelled）**
   - キャンセルされた特典の履歴

3. **期限切れ（expired）**
   - 有効期限切れの特典（今後対応予定）

### 削除される特典

- **available**: 交換可能状態
- **pending**: 交換申請済み（受付確認待ち）
