```typescript
function formatMetadata(meta: Record<string, unknown> | null): string {
  if (!meta || Object.keys(meta).length === 0) return "—";
  const parts: string[] = [];

  // 既存のフォーマット
  if (meta.current_stamp_count !== undefined) parts.push(`スタンプ: ${meta.current_stamp_count}`);
  if (meta.stamps_added !== undefined) parts.push(`+${meta.stamps_added}`);
  if (meta.type !== undefined) parts.push(String(meta.type));

  // 🆕 QRスキャン関連の追加
  if (meta.device_type !== undefined) parts.push(`デバイス: ${meta.device_type}`);
  if (meta.qr_parsed !== undefined) {
    const parsed = meta.qr_parsed as any;
    parts.push(`QR: ${parsed.type || '?'} ${parsed.stamps || '?'}個`);
  }
  if (meta.error_type !== undefined) parts.push(`エラー: ${meta.error_type}`);
  if (meta.http_status !== undefined) parts.push(`HTTP ${meta.http_status}`);

  // 既存のフォールバック
  if (meta.result !== undefined) parts.push(String(meta.result));
  if (meta.stamps_won !== undefined) parts.push(`獲得: ${meta.stamps_won}`);
  if (meta.from_page !== undefined) parts.push(`from: ${meta.from_page}`);
  if (meta.error !== undefined) parts.push(`err: ${String(meta.error).slice(0, 20)}`);

  if (parts.length > 0) return parts.join(", ");
  return JSON.stringify(meta).slice(0, 60) + (JSON.stringify(meta).length > 60 ? "…" : "");
}
```

**効果**:
- テーブルのメタデータ列で、`デバイス: iPhone`, `QR: premium 15個`, `エラー: duplicate_scan` のように見やすく表示される
- 詳細モーダルでは従来通り完全なJSONを確認可能

## 💡 提案されたSQLの実行例

### 例1: 特定ユーザーのQRスキャンフロー追跡

Supabase SQL Editor で実行：

```sql
-- ユーザーIDまたは会員証番号で検索
SELECT
  el.created_at,
  el.event_name,
  el.metadata->>'device_type' as device,
  el.metadata->>'qr_raw_value' as qr_raw,
  el.metadata->>'request_stamps' as requested_stamps,
  el.metadata->>'stamps_added' as actual_stamps,
  el.metadata->>'error' as error,
  el.metadata
FROM event_logs el
JOIN profiles p ON el.user_id = p.id
WHERE p.ticket_number = '123456'  -- 会員証番号
  AND el.event_name LIKE 'stamp_scan%'
  AND el.created_at >= current_date - interval '7 days'
ORDER BY el.created_at DESC;
```

### 例2: iPhoneでの問題発生率

```sql
-- デバイス別のスキャン成功率
SELECT
  metadata->>'device_type' as device,
  COUNT(*) FILTER (WHERE event_name = 'stamp_scan_success') as success_count,
  COUNT(*) FILTER (WHERE event_name = 'stamp_scan_fail') as fail_count,
  ROUND(
    100.0 * COUNT(*) FILTER (WHERE event_name = 'stamp_scan_success') /
    NULLIF(COUNT(*), 0), 2
  ) as success_rate_percent
FROM event_logs
WHERE event_name IN ('stamp_scan_success', 'stamp_scan_fail')
  AND created_at >= current_date - interval '7 days'
GROUP BY metadata->>'device_type'
ORDER BY success_count DESC;
```
