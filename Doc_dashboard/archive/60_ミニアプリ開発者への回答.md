## 📋 確認事項への回答

### 1. 設計方針の確認 ✅

**質問: 「自動付与式」の設計意図について**

> **ミニアプリ側の理解:**
> - 100スタンプ到達時: POIC殺菌剤**のみ**が自動付与される
> - 歯ブラシ（10の倍数）は優先度が低いため**スキップされる**
> - ユーザーは特典を**選べない**（優先度ルールに従う）

**回答:**

**✅ はい、この理解で完全に正しいです。**

#### 詳細な説明

1. **100スタンプ到達時の動作**
   - 該当するマイルストーン:
     - 10の倍数（100） → 歯ブラシ（優先度: 1）
     - 50の倍数（100） → POIC（優先度: 2）
   - **付与される特典:** POIC殺菌剤のみ（優先度が高い方）
   - **付与されない:** 歯ブラシ（優先度ルールによりスキップ）

2. **ユーザーが特典を選べないことについて**
   - **はい、これは意図した設計です**
   - 理由:
     - システムの複雑性を減らすため
     - 在庫管理を簡素化するため
     - より価値の高い特典を優先的に提供するため

3. **スキップされた特典について**
   - **後から交換することはできません**
   - Doc 55:113-115に明記:
     ```
     ⚠️ 重要:
     - 特典は到達時に1つのみ付与されます
     - 優先度が高い特典のみが付与され、低い特典はスキップされます
     - スキップされた特典は次のマイルストーンでも獲得できません
     ```

#### 優先度の設計思想

```
選べる自費メニュー（優先度: 3） > POIC殺菌剤（優先度: 2） > 歯ブラシ（優先度: 1）
```

- **自費メニュー:** 高額な施術の割引（最も価値が高い）
- **POIC:** 本体込みまたは補充用（中程度の価値）
- **歯ブラシ:** 物品プレゼント（基本的な特典）

**→ この設計で進めてください**

### 2. 在庫管理の方針 ✅

**質問: 自動付与時の在庫確認について**

> **ミニアプリ側の懸念:**
> - スタンプ付与時に自動的に `reward_exchanges` にレコードが作成される
> - この時点では在庫確認ができない
> - POIC殺菌剤や歯ブラシの在庫が切れても特典が発行される
> - 受付で「在庫切れです」と言わざるを得ない状況が発生する可能性

**回答:**

**✅ この運用は許容できます。以下の方針で進めてください。**

#### 在庫管理方針

1. **ミニアプリ側では在庫チェック不要**
   - マイルストーン到達時に無条件で `reward_exchanges` にレコードを作成してください
   - `status = 'pending'` で特典を付与

2. **在庫管理は管理画面側で行います**
   - 管理画面で `reward_exchanges` の `pending` レコードを確認
   - 在庫がない場合、受付スタッフが以下の対応を行います:
     - **Option A:** 次回来院時に引き渡すことを案内
     - **Option B:** 別の特典と交換を提案
     - **Option C:** 手動でキャンセル処理（`status = 'cancelled'`, notes = '在庫切れ'）

3. **在庫切れの場合の運用フロー**
   ```
   1. ユーザーがアプリで「未受け取り特典」を確認
   2. 受付で特典を申し出る
   3. スタッフが在庫を確認
   4. 在庫切れの場合:
      - 管理画面で当該 reward_exchange を開く
      - キャンセルボタンをクリック
      - notes に「在庫切れ、次回来院時に引き渡し」と記載
   5. ユーザーに説明
   ```

4. **将来的な改善案（オプション）**
   - 管理画面に「在庫数アラート」機能を追加
   - 在庫が少なくなったらスタッフに通知
   - ただし、現時点では実装不要

**→ ミニアプリ側は在庫チェックなしで特典を自動付与してください**

#### 移行ボーナスの詳細

**1. 移行ボーナスの内容**

```
特典: 歯ブラシ 1本
有効期限: 当日限り
対象: スタンプ数 > 0 のすべてのユーザー
```

理由:
- 歯ブラシは在庫管理が比較的容易
- 当日限りなので、来院時に確実に引き渡せる
- シンプルで分かりやすい

**2. 付与タイミング**

```
マイグレーション実行直後（021, 022の後）
```

**3. 実施SQL**

Doc 59の移行ボーナス付与SQLを使用します:

```sql
-- 既存ユーザー（スタンプ数 > 0）に移行ボーナスを付与
INSERT INTO reward_exchanges (
  user_id,
  reward_id,
  milestone_reached,
  status,
  valid_until,
  is_first_time,
  is_milestone_based,
  stamp_count_used,
  notes,
  exchanged_at
)
SELECT
  p.id AS user_id,
  (SELECT id FROM milestone_rewards WHERE reward_type = 'toothbrush' LIMIT 1) AS reward_id,
  -1 AS milestone_reached, -- 特別値（移行ボーナス）
  'pending' AS status,
  (NOW() + INTERVAL '23 hours 59 minutes')::TIMESTAMPTZ AS valid_until, -- 当日限り
  false AS is_first_time,
  true AS is_milestone_based,
  0 AS stamp_count_used,
  '特典システム移行記念ボーナス（歯ブラシ1本）- 当日限り有効' AS notes,
  NOW() AS exchanged_at
FROM profiles p
WHERE p.stamp_count > 0;

-- 確認クエリ
SELECT COUNT(*) FROM reward_exchanges WHERE milestone_reached = -1;
```

**4. ユーザーへの通知**

移行ボーナス付与後、LINE通知を送信:

```

さらに、システム移行記念として
全員に「歯ブラシ 1本」をプレゼント🎁
```

## 🎯 ミニアプリ側の実装に関する追加情報

### 実装推奨事項

#### 1. 型定義の方針

**推奨: Option B（新しい型を作成）**

```typescript
// types/reward.ts

// 旧仕様（手動交換型）
export interface LegacyReward {
  id: string;
  name: string;
  description: string | null;
  required_stamps: number;
  // ...
}

// 新仕様（マイルストーン型）
export interface MilestoneReward {
  id: string;
  name: string;
  description: string | null;
  milestone_type: 'every_10' | 'every_50' | 'every_150_from_300';
  validity_months: number | null; // 0 = 当日限り, NULL = 無期限, N = Nヶ月
  reward_type: 'toothbrush' | 'poic' | 'premium_menu';
  // ...
}
```

理由:
- 新旧を明確に分離
- TypeScriptの型安全性を維持
- 将来的に `rewards` テーブルを削除できる

#### 2. API エンドポイントの方針

**推奨: 新規エンドポイントを作成**

```
GET /api/rewards             → 旧仕様（互換性のため維持）
GET /api/milestone-rewards   → 新仕様
POST /api/rewards/exchange   → 旧仕様（互換性のため維持）
```

#### 3. マイルストーン判定の実装

**重要なポイント:**

```typescript
// lib/milestones.ts

// 有効期限の計算
function calculateValidUntil(rewardType: RewardType): Date | null {
  if (rewardType === RewardType.TOOTHBRUSH) {
    // 歯ブラシ: 当日限り（翌日0時まで）
    const validUntil = new Date();
    validUntil.setHours(23, 59, 59, 999);
    return validUntil;
  } else if (rewardType === RewardType.POIC || rewardType === RewardType.PREMIUM_MENU) {
    // POIC・自費メニュー: 5ヶ月間
    return addMonths(new Date(), 5);
  }
  return null;
}
```

**注意:**
- 歯ブラシは `validity_months = 0` だが、valid_until には当日の23:59:59を設定
- POIC・自費メニューは正確に5ヶ月後を計算
