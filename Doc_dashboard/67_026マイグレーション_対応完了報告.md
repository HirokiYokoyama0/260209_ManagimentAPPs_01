## 3. 技術的変更内容

### Before（修正前）
```typescript
import { createSupabaseServerClient } from "@/lib/supabase/server";

export async function DELETE(request: NextRequest) {
  const supabase = await createSupabaseServerClient(); // ANON_KEY使用

  const { error } = await supabase
    .from("reward_exchanges")
    .delete()
    .eq("id", id); // RLSでブロックされる
}
```

### After（修正後）
```typescript
import { createSupabaseAdminClient } from "@/lib/supabase/server-admin";

export async function DELETE(request: NextRequest) {
  const supabase = createSupabaseAdminClient(); // SERVICE_ROLE_KEY使用

  const { error } = await supabase
    .from("reward_exchanges")
    .delete()
    .eq("id", id); // RLSをバイパス - 正常動作
}
```

### 変更ポイント
1. **import文変更**: `server` → `server-admin`
2. **関数呼び出し変更**: `await createSupabaseServerClient()` → `createSupabaseAdminClient()`
3. **認証キー変更**: ANON_KEY → SERVICE_ROLE_KEY
4. **RLS適用**: 適用される → バイパスされる

## 8. トラブルシューティング

### もし本番環境でエラーが発生した場合

#### パターン1: 削除・更新エラー
```
Error: new row violates row-level security policy
```

**原因**: APIがまだServerClientを使用している
**対処**: 該当APIを `createSupabaseAdminClient()` に変更

#### パターン2: 認証エラー
```
Error: Missing SERVICE_ROLE_KEY
```

**原因**: 環境変数 `SUPABASE_SERVICE_ROLE_KEY` が未設定
**対処**: Vercelの環境変数設定を確認
