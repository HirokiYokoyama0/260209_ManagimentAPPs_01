### 質問1: 使用しているAPIエンドポイント

**回答**: **B. 管理ダッシュボード専用の別のAPIエンドポイントを使用**

管理ダッシュボードでは以下の2つのAPIエンドポイントを使用しています:

#### 1. スタンプ増減API
- **エンドポイント**: `PATCH /api/profiles/{id}/stamp`
- **ファイル**: `app/api/profiles/[id]/stamp/route.ts`
- **用途**: スタンプ数を相対的に変更（+1, -1など）
- **呼び出し箇所**: `components/admin/patients-table.tsx:121-132` の `updateStampDelta()`

```typescript
// フロントエンド (患者一覧テーブル)
async function updateStampDelta(id: string, delta: number) {
  const res = await fetch(`/api/profiles/${id}/stamp`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ delta }),
  });
  // ...
}
```

#### 2. スタンプ直接設定API ⭐ **問題の原因**
- **エンドポイント**: `PATCH /api/profiles/{id}/stamp-set`
- **ファイル**: `app/api/profiles/[id]/stamp-set/route.ts`
- **用途**: スタンプ数を絶対値で設定（例: 0, 15など）
- **呼び出し箇所**: `components/admin/patients-table.tsx:134-146` の `setStampCount()`

```typescript
// フロントエンド (スタンプ数変更ダイアログ)
async function setStampCount(id: string, stamp_count: number) {
  const res = await fetch(`/api/profiles/${id}/stamp-set`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ stamp_count }),
  });
  // ...
}
```

**重要**: LIFFアプリ側の `/api/stamps/manual` は**使用していません**。

#### `handleStampMilestones()` の動作

**ファイル**: `lib/milestones.ts:261-296`

```typescript
export async function handleStampMilestones(
  userId: string,
  oldStampCount: number,
  newStampCount: number
) {
  // 1. マイルストーンを判定
  const milestones = checkMilestones(oldStampCount, newStampCount);

  if (milestones.length === 0) {
    console.log(`ℹ️ マイルストーン到達なし: ${oldStampCount} → ${newCount}`);
    return [];
  }

  console.log(`🎯 マイルストーン到達: ${milestones.length}件`, milestones);

  // 2. 各マイルストーンに対して特典を付与
  const grantedRewards = [];

  for (const { milestone, rewardType } of milestones) {
    try {
      const exchange = await grantMilestoneReward(userId, milestone, rewardType);
      grantedRewards.push({
        milestone,
        rewardType,
        exchange
      });
    } catch (error) {
      console.error(`❌ マイルストーン特典付与エラー (${milestone}, ${rewardType}):`, error);
      // エラーでも続行（他のマイルストーンは処理する）
    }
  }

  console.log(`✅ マイルストーン処理完了: ${grantedRewards.length}件付与`);

  return grantedRewards;
}
```

**動作**:
- スタンプ数が**増加**した場合: マイルストーン判定を実行し、特典を**自動付与**
- スタンプ数が**減少**した場合: マイルストーン判定は**スキップ**（到達なし）

### 質問3: スタンプ減少時の挙動

**回答**: **いいえ、実装されていません** ❌ **これも問題の一因です**

#### 現在の実装

両方のAPIで `invalidateMilestoneRewards()` 関数は**呼び出していません**。

スタンプ減少時の処理:

##### `stamp/route.ts` (行44-63)
```typescript
// ⚠️ 重要：スタンプ数を減らす場合、過去の大きい値を削除
if (newCount < currentStampCount) {
  console.log(`🗑️ スタンプ数を減らすため、stamp_number > ${newCount} のレコードを削除`);

  // SERVICE_ROLE_KEY で削除（ANON_KEY では RLS により DELETE 不可）
  const { error: deleteError, count } = await supabaseAdmin
    .from("stamp_history")
    .delete({ count: "exact" })
    .eq("user_id", id)
    .gt("stamp_number", newCount);

  if (deleteError) {
    console.error("❌ 過去レコード削除エラー:", deleteError);
    return NextResponse.json(
      { error: `過去レコード削除失敗: ${deleteError.message}` },
      { status: 500 }
    );
  }

  console.log(`✅ ${count}件 の過去レコードを削除しました`);
}
```

**問題点**:
- `stamp_history` テーブルの過去レコードは削除している
- しかし、`reward_exchanges` テーブルのマイルストーン特典は**削除・無効化していない**
- そのため、スタンプを0にしても特典が `pending` のまま残る

## 🔍 問題の根本原因分析

### 問題1: スタンプ減少時の特典残存

**原因**: `invalidateMilestoneRewards()` 関数を呼んでいない

**現在の動作**:
1. スタッフが管理ダッシュボードでスタンプ数を15 → 0に変更
2. `stamp_history` の過去レコードは削除される
3. `profiles.stamp_count` は0に更新される
4. しかし、`reward_exchanges` の特典（milestone: 10）は `pending` のまま残る

**期待される動作**:
- `reward_exchanges` の status を `'cancelled'` に変更すべき

---

### 問題2: スタンプ増加時の自動付与

**原因**: `handleStampMilestones()` 関数を呼んでいる

**現在の動作**:
1. スタッフが管理ダッシュボードでスタンプ数を0 → 15に変更
2. `handleStampMilestones(id, 0, 15)` が実行される
3. マイルストーン10を検出
4. `grantMilestoneReward()` が実行され、特典が `pending` で自動付与される

**期待される動作** (LIFFアプリ側の仕様):
- スタッフ操作では自動付与**しない**

---

### 問題3: マイルストーン特典の削除失敗

**原因**: 問題1と同じ（`invalidateMilestoneRewards()` 関数を呼んでいない）

## 💡 LIFFアプリ側との実装の違い

### LIFFアプリ側の実装 (`/api/stamps/manual`)

```typescript
// スタンプ減少時
if (changeAmount < 0) {
  await invalidateMilestoneRewards(userId, currentStampCount, newStampCount);
}

// スタンプ増加時
if (changeAmount > 0) {
  // 何もしない（マイルストーン判定も特典付与もしない）
}
```

### 管理ダッシュボード側の実装 (`/api/profiles/{id}/stamp-set`)

```typescript
// スタンプ減少時
if (newCount < currentStampCount) {
  // stamp_history の削除のみ
  // ❌ invalidateMilestoneRewards() を呼んでいない
}

// スタンプ増加時
// ⚠️ handleStampMilestones() を呼んでいる（自動付与される）
milestones = await handleStampMilestones(id, currentStampCount, newCount);
```

**違い**:
- LIFFアプリ側: スタンプ増加時に自動付与**しない**
- 管理ダッシュボード側: スタンプ増加時に自動付与**する**

## 🎯 推奨される修正方針

### 方針A: LIFFアプリ側の実装に統一（推奨）⭐

**利点**:
- スタッフ操作時の挙動が統一される
- 意図しない特典付与を防げる
- マイルストーン特典の無効化処理が正しく動作する

**修正内容**:

#### 1. `stamp/route.ts` の修正

```typescript
// 修正前

  milestones = await handleStampMilestones(id, currentStampCount, newCount);
  // ...
}

// 修正後
// 6️⃣ スタンプ減少時のマイルストーン特典無効化
if (newCount < currentStampCount) {
  try {
    console.log(`🔄 スタンプ減少を検出: ${currentStampCount} → ${newCount}`);
    await invalidateMilestoneRewards(id, currentStampCount, newCount);
    console.log(`✅ マイルストーン特典を無効化しました`);
  } catch (error) {
    console.error("❌ マイルストーン無効化エラー:", error);
    // エラーでもスタンプ減少は成功しているので続行
  }
```
