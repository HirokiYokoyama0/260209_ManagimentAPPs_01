# スタッフ操作時のマイルストーン特典問題 - 詳細分析と修正方針

**作成日**: 2026-04-04
**作成者**: 管理ダッシュボード開発チーム (Claude Code)
**緊急度**: 🚨 **高（本番環境に影響あり）**

## 🚨 最重要問題: ステータスが間違っている

### 問題の核心

**データベース仕様** ([05_Database_Schema.md:379](Doc_dashboard/05_Database_Schema.md))
```markdown
| ステータス | 意味 | 運用 |
|-----------|------|------|
| `pending` | 受付で確認中 | 特典を提供する前 |
| `available` | この特典と交換する | ユーザーが交換申請可能な状態 |  ← **これが正しい**
| `completed` | 提供完了 | 受付で実際に特典を渡した後 |
| `cancelled` | キャンセル | 誤交換などの取り消し |
| `expired` | 期限切れ | 有効期限切れ（新仕様のみ） |
```

**現在の実装** (`lib/milestones.ts:215`)
```typescript
status: 'pending',  // ❌ 間違い！
```

**問題点**:
- マイルストーン到達時に `status: 'pending'`（受付で確認中）として作成される
- **これは完全に間違っている**
- ユーザーは「この特典と交換する」ボタンをタップできない
- 受付で特典が出せない（「受付確認中」のままになる）

**期待される動作**:
1. マイルストーン到達時: `status: 'available'`（この特典と交換する）
2. ユーザーが「この特典と交換する」をタップ: `status: 'pending'`（受付で確認中）
3. 受付で特典を渡す: `status: 'completed'`（提供完了）

**影響範囲**:
- LIFFアプリでマイルストーン到達した全ユーザー
- 管理ダッシュボードでスタンプを追加した全ユーザー
- **本番環境で既に発生している可能性が高い** ⚠️

---

## 📊 問題の全体像

管理ダッシュボード側には **3つの重大な問題** があります:

| 問題 | 現在の動作 | 期待される動作 | 影響度 |
|-----|-----------|--------------|--------|
| **問題1: ステータスが `pending`** | `status: 'pending'` で作成 | `status: 'available'` で作成 | 🔴 **最重要** |
| **問題2: スタンプ増加時に自動付与** | マイルストーン特典を自動付与 | 自動付与**しない** | 🟡 中 |
| **問題3: スタンプ減少時に特典が残る** | 特典が `pending` のまま残る | 特典を `'cancelled'` に変更 | 🟡 中 |

## 🔍 詳細分析

### 問題1: ステータスが `'pending'` ⚠️ **最重要**

#### 根本原因

**ファイル**: `lib/milestones.ts:185-224`

```typescript
export async function grantMilestoneReward(
  userId: string,
  milestone: number,
  rewardType: RewardType
) {
  const supabase = await createSupabaseServerClient();

  // 1. 特典タイプに応じてマスターデータを取得
  const reward = await getMilestoneReward(rewardType);

  if (!reward) {
    throw new Error(`Reward not found for type: ${rewardType}`);
  }

  // 2. 初回判定（POIC用）
  let isFirstTime = false;
  if (rewardType === RewardType.POIC) {
    isFirstTime = await checkIfFirstTimePOIC(userId, reward.id);
  }

  // 3. 有効期限計算
  const validUntil = calculateValidUntil(rewardType);

  // 4. 特典付与レコード作成
  const { data: exchange, error } = await supabase
    .from('reward_exchanges')
    .insert({
      user_id: userId,
      reward_id: reward.id,
      milestone_reached: milestone,
      status: 'pending',  // ❌ ここが間違い！'available' にすべき
      valid_until: validUntil?.toISOString(),
      is_first_time: isFirstTime,
      is_milestone_based: true,
      stamp_count_used: milestone, // 参考値
      notes: `${milestone}スタンプ到達で自動付与（${rewardType}）`,
      exchanged_at: new Date().toISOString()
    })
    .select()
    .single();

  // ...
}
```

#### 修正内容

```typescript
status: 'available',  // ✅ 修正
```

#### この問題の深刻さ

- **既存データへの影響**: 本番環境で既に作成された特典レコードが全て `'pending'` になっている可能性
- **ユーザー体験への影響**: 「この特典と交換する」ボタンが表示されない
- **受付業務への影響**: 特典の状態が分からない

### 問題3: スタンプ減少時に特典が残る

**ファイル**: `app/api/profiles/[id]/stamp/route.ts:44-63`

```typescript
// ⚠️ 重要：スタンプ数を減らす場合、過去の大きい値を削除
if (newCount < currentStampCount) {
  console.log(`🗑️ スタンプ数を減らすため、stamp_number > ${newCount} のレコードを削除`);

  // SERVICE_ROLE_KEY で削除（ANON_KEY では RLS により DELETE 不可）
  const { error: deleteError, count } = await supabaseAdmin
    .from("stamp_history")
    .delete({ count: "exact" })
    .eq("user_id", id)
    .gt("stamp_number", newCount);

  if (deleteError) {
    console.error("❌ 過去レコード削除エラー:", deleteError);
    return NextResponse.json(
      { error: `過去レコード削除失敗: ${deleteError.message}` },
      { status: 500 }
    );
  }

  console.log(`✅ ${count}件 の過去レコードを削除しました`);
}

// ❌ reward_exchanges テーブルの特典は残ったまま！
```

```sql
-- ステータスが 'pending' のマイルストーン特典を確認

  id,
  user_id,
  milestone_reached,
  status,
  created_at
FROM reward_exchanges
WHERE is_milestone_based = true
  AND status = 'pending'
```
