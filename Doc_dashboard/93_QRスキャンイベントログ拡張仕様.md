# QRスキャンイベントログ拡張仕様

**作成日**: 2026-03-28
**目的**: スタンプ個数不具合などの問題調査を可能にする
**優先度**: 🔴 高（iPhone問題の原因特定に必要）

## 📋 背景・目的

### 現在の問題

iPhoneテストで以下の不具合が報告されている：
1. **15スタンプQRが10ポイントしか付与されない**
2. **購買インセンティブQRが1度しかスキャンできない**

現在のイベントログでは**スタンプ個数しか記録されていない**ため、以下の情報が不足：
- QRコード生値（スキャンしたペイロード）
- パース後の値
- デバイス情報（iPhone/Android判定）
- APIリクエスト/レスポンスの詳細
- エラーの詳細

### 実現したいこと

✅ **QRコード生値の記録** - ペイロード自体が間違っているか確認
✅ **パース前後の比較** - LIFF scanCodeV2の挙動差を検証
✅ **デバイス情報** - iPhone特有の問題か判定
✅ **API通信の追跡** - リクエスト→レスポンスの流れを完全追跡
✅ **エラーの詳細記録** - どこで失敗したか特定

## 🎯 設計方針

### 1. 既存実装を活用

既に `lib/analytics.ts` に以下が実装済み：
- `logStampScanSuccess()` - 成功時
- `logStampScanFail()` - 失敗時

→ **これらを拡張**して詳細情報を記録

### 2. 記録タイミング

```
[フロントエンド: app/scan/page.tsx]
  ① QRスキャン開始 (stamp_scan_start)
    ↓
  ② scanCodeV2 実行 → QR生値取得
    ↓
  ③ JSONパース
    ↓
  ④ APIリクエスト送信 (stamp_scan_api_request) ← NEW!
    ↓
[バックエンド: app/api/stamps/scan/route.ts]
  ⑤ リクエスト受信
    ↓
  ⑥ バリデーション
    ↓
  ⑦ DB操作
    ↓
  ⑧ 成功 or 失敗
    ↓
  ⑨ イベントログ送信 (stamp_scan_success or stamp_scan_fail)
    ↓
[フロントエンド]
  ⑩ レスポンス受信
    ↓
  ⑪ 結果表示
```

### 1. lib/analytics.ts の拡張

#### 新規関数: logStampScanStart
```typescript
/**
 * QRスキャン開始ログ（デバイス情報付き）
 */
export const logStampScanStart = (params: {
  userId?: string;
  deviceType?: 'iPhone' | 'Android' | 'Unknown';
  metadata?: Record<string, any>;
}) => {
  return logEvent({
    eventName: 'stamp_scan_start',
    userId: params.userId,
    metadata: {
      ...params.metadata,
      device_type: params.deviceType,
      liff_version: typeof window !== 'undefined' && window.liff ? window.liff.getVersion() : null,
    },
  });
};
```

#### 新規関数: logStampScanApiRequest
```typescript
/**
 * QRスキャンAPIリクエストログ（QR生値とパース後の値を記録）
```

```typescript
// QRスキャン開始
const handleScanQR = async () => {
  // デバイス判定
  const deviceType = /iPhone|iPad|iPod/.test(navigator.userAgent) ? 'iPhone' :
                     /Android/.test(navigator.userAgent) ? 'Android' : 'Unknown';

  // ログ送信
  await logStampScanStart({ userId: profile?.userId, deviceType });

  try {
    const result = await window.liff.scanCodeV2();
    await processQRCode(result.value);
  } catch (err) {
    // エラー処理...
  }
};

// QRコード処理
const processQRCode = async (qrValue: string) => {
  // QR生値をログに記録
  console.log('📱 [Scan Page] QR生値:', qrValue);

  // JSONパース
  let payload: QRPayload;
  try {
    payload = JSON.parse(qrValue);
    console.log('📱 [Scan Page] パース後:', payload);
  } catch {
    setError('無効なQRコードです');
    return;
  }

  // APIリクエストペイロード作成
  const qrCodeId = payload.type === 'purchase'
    ? `${payload.type}_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`
    : `${payload.type}_${new Date().toISOString().split('T')[0]}`;

  const requestPayload = {
    userId: profile!.userId,
    type: payload.type,
    stamps: payload.stamps,
    qrCodeId,
  };

  // 🆕 APIリクエスト前にログ送信
  await logStampScanApiRequest({
    userId: profile?.userId,
    qrRawValue: qrValue,
    qrParsed: payload,
    requestPayload,
  });

  // API呼び出し
  const response = await fetch('/api/stamps/scan', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(requestPayload),
  });

  const data = await response.json();
  // レスポンス処理...
};
```

## 🔍 調査用SQL

### 1. 特定ユーザーのQRスキャンフロー追跡

```sql
-- ユーザーIDまたは会員証番号で検索
SELECT
  el.created_at,
  el.event_name,
  el.metadata->>'device_type' as device,
  el.metadata->>'qr_raw_value' as qr_raw,
  el.metadata->>'request_stamps' as requested_stamps,
  el.metadata->>'stamps_added' as actual_stamps,
  el.metadata->>'error' as error,
  el.metadata
FROM event_logs el
JOIN profiles p ON el.user_id = p.id
WHERE p.ticket_number = '123456'  -- 会員証番号
  AND el.event_name LIKE 'stamp_scan%'
  AND el.created_at >= current_date - interval '7 days'
ORDER BY el.created_at DESC;
```

### 2. QR生値とパース結果の比較

```sql
-- QRコード生値と実際に付与されたスタンプ数を比較
WITH qr_requests AS (
  SELECT
    user_id,
    created_at,
    metadata->>'qr_raw_value' as qr_raw,
    (metadata->'qr_parsed'->>'stamps')::int as parsed_stamps,
    (metadata->'request_payload'->>'stamps')::int as request_stamps
  FROM event_logs
  WHERE event_name = 'stamp_scan_api_request'
    AND created_at >= current_date - interval '7 days'
),
qr_results AS (
  SELECT
    user_id,
    created_at,
    (metadata->>'stamps_added')::int as actual_stamps
  FROM event_logs
  WHERE event_name = 'stamp_scan_success'
    AND created_at >= current_date - interval '7 days'
)
SELECT
  p.display_name,
  req.created_at,
  req.qr_raw,
  req.parsed_stamps,
  req.request_stamps,
  res.actual_stamps,
  CASE
    WHEN req.request_stamps != res.actual_stamps THEN '⚠️ 不一致'
    ELSE '✅ 一致'
  END as status
FROM qr_requests req
LEFT JOIN qr_results res
  ON req.user_id = res.user_id
  AND res.created_at BETWEEN req.created_at AND req.created_at + interval '5 seconds'
JOIN profiles p ON req.user_id = p.id
ORDER BY req.created_at DESC;
```

### 3. iPhoneでの問題発生率

```sql
-- デバイス別のスキャン成功率
SELECT
  metadata->>'device_type' as device,
  COUNT(*) FILTER (WHERE event_name = 'stamp_scan_success') as success_count,
  COUNT(*) FILTER (WHERE event_name = 'stamp_scan_fail') as fail_count,
  ROUND(
    100.0 * COUNT(*) FILTER (WHERE event_name = 'stamp_scan_success') /
    NULLIF(COUNT(*), 0), 2
  ) as success_rate_percent
FROM event_logs
WHERE event_name IN ('stamp_scan_success', 'stamp_scan_fail')
  AND created_at >= current_date - interval '7 days'
GROUP BY metadata->>'device_type'
ORDER BY success_count DESC;
```

### 4. スタンプ個数不一致の検出

```sql
-- リクエストとレスポンスでスタンプ数が異なるケースを抽出
WITH requests AS (
  SELECT
    user_id,
    created_at,
    metadata->>'qr_raw_value' as qr_raw,
    (metadata->'request_payload'->>'stamps')::int as requested_stamps,
    metadata->>'device_type' as device
  FROM event_logs
  WHERE event_name = 'stamp_scan_api_request'
    AND created_at >= current_date - interval '7 days'
),
successes AS (
  SELECT
    user_id,
    created_at,
    (metadata->>'stamps_added')::int as actual_stamps
  FROM event_logs
  WHERE event_name = 'stamp_scan_success'
    AND created_at >= current_date - interval '7 days'
)
SELECT
  p.display_name,
  p.ticket_number,
  req.created_at,
  req.device,
  req.qr_raw,
  req.requested_stamps,
  suc.actual_stamps,
  (req.requested_stamps - suc.actual_stamps) as差分
FROM requests req
JOIN successes suc
  ON req.user_id = suc.user_id
  AND suc.created_at BETWEEN req.created_at AND req.created_at + interval '5 seconds'
JOIN profiles p ON req.user_id = p.id
WHERE req.requested_stamps != suc.actual_stamps  -- 不一致のみ
ORDER BY req.created_at DESC;
```

### 2. 再現性の確認

```sql
-- 同じユーザーが同じ問題を複数回経験しているか
SELECT user_id, COUNT(*) as occurrence_count
FROM event_logs
WHERE event_name = 'stamp_scan_api_request'
  AND (metadata->'request_payload'->>'stamps')::int != 15
  AND metadata->>'qr_raw_value' LIKE '%"stamps":15%'
GROUP BY user_id
HAVING COUNT(*) > 1;
```

### 3. デバイス依存性の検証

```sql
-- iPhone でのみ発生しているか確認
SELECT
  metadata->>'device_type' as device,
  COUNT(*) as issue_count
FROM event_logs
WHERE event_name = 'stamp_scan_api_request'
  AND (metadata->'request_payload'->>'stamps')::int != (metadata->'qr_parsed'->>'stamps')::int
GROUP BY device;
```
