## 🎯 原因の仮説

### 仮説1: LIFF初期化の無限ループ

```typescript
// 想定される問題コード
liff.init({ liffId: LIFF_ID })
  .catch(error => {
    console.error('LIFF init failed:', error);
    // エラー時に再初期化を試みている？
    liff.init({ liffId: LIFF_ID }); // 無限ループの可能性
  });
```

**対策:**
- リトライ回数の上限を設定
- エラー時は明示的にエラー画面を表示し、ユーザーに再試行を促す

### 仮説2: ブラウザ ⇔ LINEアプリの切り替えループ

1. 登録QRのLIFF URLを開く
2. 外部ブラウザで開かれる（iOS 18の仕様変更？）
3. LINEアプリに戻そうとする（`window.location.href = 'line://...'`）
4. 再度外部ブラウザで開かれる
5. **無限ループ**

**対策:**
- `liff.isInClient()` で環境を判定
- 外部ブラウザの場合は専用のエラーメッセージを表示

### 仮説3: iOS 18のWebView制限

iOS 16/17/18でWebViewの動作が変更され、LIFF SDKが未対応の可能性。

**対策:**
- LIFF SDKを最新版にアップデート
- LINE公式のiOS 18互換性情報を確認

## 🔧 確認・修正依頼事項

### 【緊急】1. LIFF初期化コードの確認

以下のコードを確認してください：

```typescript
// app/layout.tsx または useEffect内のLIFF初期化処理
liff.init({ liffId: LIFF_ID })
  .then(() => {
    // 初期化成功
  })
  .catch(error => {
    // エラーハンドリングの確認
    // ここで再度init()を呼んでいないか？
  });
```

#### チェックポイント:
- [ ] エラー時に`liff.init()`を再度呼んでいないか
- [ ] リトライロジックがある場合、上限回数が設定されているか

useEffect(() => {
  liff.init({ liffId: LIFF_ID })
    .then(() => {
      // LIFF環境で動作しているか確認

        // 外部ブラウザで開かれた場合

    });
