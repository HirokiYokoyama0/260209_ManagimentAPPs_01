### 2026-04-04
- **🟣 availableステータス対応: TypeScript型定義とUI更新**
  - **背景**
    - LIFFアプリ開発者が 'available' ステータスを導入（マイルストーン到達時の初期状態）
    - 管理ダッシュボード側のTypeScript型定義が古く、コンパイルエラーが発生
    - 115番ドキュメントの警告通り、型定義とUIコンポーネントの更新が必要
  - **実装内容**
    1. **型定義の更新（lib/types.ts）**
       - `RewardExchange.status` に 'available' を追加
       - 順序: `'available' | 'pending' | 'completed' | 'cancelled' | 'expired'`
    2. **モバイルカード対応（mobile-reward-exchange-card.tsx）**
       - statusConfig に 'available' を追加
       - ラベル: 「交換可能」
       - カラー: 紫色（bg-purple-100 text-purple-800）
    3. **特典交換履歴ページ対応（reward-exchanges/page.tsx）**
       - getStatusBadge() に 'available' ケースを追加
       - フィルターに「交換可能のみ」オプションを追加
       - 'available' ステータスでも操作ボタン（完了/キャンセル/削除）を表示
    4. **モバイルアクション対応（mobile-reward-exchange-actions.tsx）**
       - Props の status 型に 'available' を追加
       - 'available' ステータスでも完了/キャンセル/削除ボタンを表示
  - **ステータスの意味**
    - `available`: マイルストーン到達時に自動付与された状態（「この特典と交換する」ボタンが表示される）
    - `pending`: 患者が「交換申請」ボタンを押した状態（受付での確認待ち）
    - `completed`: スタッフが引き渡し完了した状態
    - `cancelled`: キャンセルされた状態
    - `expired`: 有効期限切れの状態
  - **修正ファイル**
    - `lib/types.ts` - 型定義に 'available' 追加
    - `components/admin/mobile/mobile-reward-exchange-card.tsx` - statusConfig 追加
    - `app/admin/reward-exchanges/page.tsx` - バッジ・フィルター・アクションボタン対応
    - `components/admin/mobile/mobile-reward-exchange-actions.tsx` - Props 型更新・ロジック修正
  - **効果**
    - TypeScriptコンパイルエラーが解消
    - LIFFアプリ側で付与された 'available' ステータスの特典が正しく表示される
    - マイルストーン到達時の特典が管理画面で「交換可能」として識別可能
  - **関連コミット**

- **🔒 スタッフ操作時のマイルストーン特典問題修正（緊急対応）**
  - **背景**

    3. **無効化問題**: スタンプ減少時に該当特典を削除すべき
  - **実装内容**
    1. **ステータス修正（lib/milestones.ts:215）**
       - `status: 'pending'` → `status: 'available'`
       - コメント追加: `// ✅ 修正: 'pending' → 'available'（この特典と交換する状態）`
    2. **自動付与を無効化**
       - `app/api/profiles/[id]/stamp/route.ts` から `handleStampMilestones()` 削除
       - `app/api/profiles/[id]/stamp-set/route.ts` から `handleStampMilestones()` 削除
       - コメント: `⚠️ スタッフ操作では自動付与しない（LIFFアプリ側の仕様に統一）`
    3. **無効化処理の実装**
       - `lib/milestones.ts` に `invalidateMilestoneRewards()` 関数を追加
       - スタンプ減少時、該当するマイルストーン特典を完全削除（DELETE）
       - 両APIエンドポイントに無効化処理を追加
    4. **RLS権限問題の緊急修正**
       - 026マイグレーション後、ANON_KEYではDELETEが禁止される
       - `invalidateMilestoneRewards()` で `createSupabaseAdminClient()` を使用（SERVICE_ROLE_KEY）
       - コメント: `✅ 修正: AdminClient を使用（026マイグレーション後はANON_KEYでUPDATEできない）`

    - `app/api/profiles/[id]/stamp/route.ts` - 自動付与削除・無効化追加

    - スタッフ操作とLIFFアプリ操作の仕様が統一
