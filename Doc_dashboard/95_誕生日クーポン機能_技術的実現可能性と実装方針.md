## ✅ 技術的実現可能性の結論

### 質問: 管理ダッシュボードからLINE公式アカウントのカードタイプメッセージを特定ユーザーに送付できるか？

**回答**: **完全に可能です。**

### 理由

1. ✅ **LINE Messaging API の基盤が既に実装済み**
   - 個別メッセージ送信機能あり ([app/api/profiles/[id]/care-messages/route.ts](../app/api/profiles/[id]/care-messages/route.ts))
   - 一斉配信機能あり ([app/api/broadcast/send/route.ts](../app/api/broadcast/send/route.ts))
   - Push Message API 連携済み ([lib/line-messaging.ts](../lib/line-messaging.ts))

2. ✅ **Flex Message（カードタイプメッセージ）のサポート**
   - LINE Messaging API は Flex Message を完全サポート
   - 現在は `type: "text"` のみだが、`type: "flex"` に変更するだけで対応可能
   - JSON形式で自由なデザインのカードを作成可能

3. ✅ **管理ダッシュボードからの送信が可能**
   - 既存の個別メッセージ送信機能を拡張
   - 一斉配信機能を活用（誕生月の全員に送信）
   - 手動送信・自動送信の両方に対応可能

## 🔧 現在の実装状況

### 1. 個別メッセージ送信機能（実装済み）

**API**: `POST /api/profiles/[id]/care-messages`

**現在の仕様**:
```typescript
// テキストメッセージのみ送信
messages: [
  {
    type: "text",
    text: personalizedMessage,
  },
]
```

**Flex Message対応後**:
```typescript
// Flex Message（カードタイプ）に変更可能
messages: [
  {
    type: "flex",
    altText: "お誕生日おめでとうございます",
    contents: {
      // Flex Message の JSON 定義
    }
  }
]
```

### 2. 一斉配信機能（実装済み）

**API**: `POST /api/broadcast/send`

**現在の機能**:
- セグメント条件でユーザー絞り込み
- 変数置換（`{name}`, `{stamp_count}` など）
- レート制限対策（50ms間隔で送信）
- 配信ログ記録

**Flex Message対応後**:
- 同じ仕組みでFlex Messageを一斉送信可能
- 誕生月セグメントで絞り込み → 自動送信

### 3. LINE Messaging API連携（実装済み）

**認証方式**:
- ✅ Channel Access Token（長期）
- ✅ Channel ID + Secret（短期トークン自動取得）

**API エンドポイント**:
- `https://api.line.me/v2/bot/message/push` - 個別送信
- `https://api.line.me/v2/bot/message/multicast` - 一斉送信（最大500人）

## 🎨 Flex Messageの技術仕様

### Flex Message とは？

LINE公式アカウントで送信できる**カードタイプのリッチメッセージ**。HTML/CSSのようにレイアウトを自由に設計可能。

### 主な特徴

| 機能 | 説明 |
|------|------|
| **カード型レイアウト** | 画像、タイトル、説明文、ボタンを含むカード |
| **アクション設定** | ボタンタップで外部リンク、LIFF起動、電話発信など |
| **カルーセル表示** | 複数のカードをスワイプ表示 |
| **動的データ埋め込み** | ユーザー名、スタンプ数などを変数で埋め込み可能 |

## 🎯 実装案の比較

### 案1: 【推奨】ユーザー属性（誕生月）による完全自動通知

#### 仕組み

1. **データ収集**
   - LIFFアプリの初回登録時またはマイページで「誕生月」を入力
   - `profiles` テーブルに `birth_month` カラム（1-12の整数）を追加

2. **自動送信バッチ**
   - 毎月1日の朝9時に自動実行（Supabase Edge Functions または Vercel Cron Jobs）
   - 「今月が誕生月のユーザー」を抽出
   - Flex Message（誕生日カード）を一斉送信

3. **クーポン付与**
   - 送信と同時に `reward_exchanges` テーブルに誕生日クーポンレコードを作成
   - 有効期限: 当月末まで（例: 3月生まれなら3月31日まで）

#### メリット

✅ **ポイント数に関係なく全員にアプローチ可能**
- 休眠患者を呼び戻す強力なきっかけ
- 「自分へのご褒美」心理を刺激

✅ **手作業ゼロの完全自動化**
- 一度設定すれば24時間365日稼働
- スタッフの負担なし

✅ **マーケティング効果が高い**
- 誕生月 = 自費診療の購買意欲が高まる時期
- 「特別感」でロイヤリティ向上

#### デメリット

⚠ **初回データ入力の手間**
- ユーザーに「誕生月」を入力してもらう必要あり
- → インセンティブ付与で解決（後述）

#### メリット

- 患者数: 500人

- 月平均: 25人に送信（300人 ÷ 12ヶ月）

- **月間売上: 約112,500円**
- **年間売上: 約135万円**

- 初期投資: ソラシドワークス様の開発工数のみ
- 年間売上: 135万円

#### 5. リスクと対策

| リスク | 対策 |
|--------|------|
| 誕生月を教えたくない患者がいる | 任意入力（強制しない） + インセンティブで登録促進 |
| クーポンの使用率が低い | 魅力的な自費メニュー選定 + 期限（当月末）で緊急性創出 |
| システム障害で送信失敗 | エラーログ監視 + 翌日再送機能 |
| LINE友だち未登録の患者 | 誕生月登録時に友だち追加を誘導 |
