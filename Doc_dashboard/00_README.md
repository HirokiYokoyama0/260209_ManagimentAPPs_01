## 読む順番（推奨）

### 新規参加者
1. **00_README.md**（このファイル）
2. **01_ファイル構成.md** — プロジェクト全体の構造把握
3. **02_クイックスタート.md** — 環境構築・起動
4. **04_Supabaseセットアップ手順.md** — DB設定
5. **03_管理ダッシュボード仕様書.md** — 機能理解
6. **10_重要_スタンプ仕様_積み上げ式.md** — 必読の基本ルール

### 機能追加・改善を行う場合
1. **03_管理ダッシュボード仕様書.md** — 現在の仕様確認
2. **05_Database_Schema.md** — DBスキーマ確認
3. **199_変更履歴.md** — 最新の実装状況
4. 該当機能の仕様書（10-50系）
5. 実装後、**199_変更履歴.md** を更新

## アーカイブについて

`archive/` フォルダには、統合・整理前のオリジナルファイルが保管されています。
通常の開発では参照不要ですが、過去の経緯を確認したい場合にご利用ください。

---

## ドキュメント更新ルール

- 機能追加・変更時は、必ず該当する仕様書を更新すること
- 重要な変更は **199_変更履歴.md** に記録すること
- ドキュメントの日付形式は `YYYY-MM-DD` で統一
- 番号体系: 00系=案内、10系=コア機能、20系=家族、30系=スタッフ/ログ、40系=アンケート、50系=LIFF、80系=デプロイ、100系=参考
