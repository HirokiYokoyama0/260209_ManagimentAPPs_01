## 概要

### システム構成

```
┌──────────────────────────────────────┐
│      LINEアプリ（ユーザー）           │
│  - LINE認証                          │
│  - LIFFアプリ起動                    │
└────────────┬─────────────────────────┘
             │ LINE User ID
             ▼
┌──────────────────────────────────────┐
│    LIFF ミニアプリ（React/TS）       │
│  - liff.init()                       │
│  - liff.getProfile()                 │
│  - Supabase ANON_KEY 使用           │
└────────────┬─────────────────────────┘
             │ ANON_KEY + RLS
             ▼
┌──────────────────────────────────────┐
│      Supabase PostgreSQL             │
│  - RLS (Row Level Security) 有効    │
│  - データベース本体                  │
└──────────────────────────────────────┘
```

### 使用技術スタック

- **フレームワーク**: React（別リポジトリ）
- **言語**: TypeScript
- **認証**: LINE LIFF SDK
- **データベース**: Supabase (ANON_KEY経由)
- **デプロイ**: Vercel / Netlify（予定）

## 認証セキュリティ

### LINE LIFF SDK による認証

#### 認証フロー

```typescript
// 1. LIFF 初期化
await liff.init({ liffId: LIFF_ID });

// 2. ログインチェック
if (!liff.isLoggedIn()) {
  liff.login();
  return;
}

// 3. プロフィール取得
const profile = await liff.getProfile();
const lineUserId = profile.userId;  // "Uxxxxxxxxxxxxxxxxxx"
const displayName = profile.displayName;
const pictureUrl = profile.pictureUrl;

// 4. Supabase に保存/更新
await supabase.from('profiles').upsert({
  id: lineUserId,
  line_user_id: lineUserId,
  display_name: displayName,
  picture_url: pictureUrl
});
```

#### セキュリティ特性

| 項目 | 状態 | 説明 |
|-----|------|------|
| 認証方式 | LINE OAuth 2.0 | ✅ LINE側が保証 |
| ユーザーID | LINE User ID | ✅ 一意で偽装不可 |
| セッション管理 | LIFF SDK | ✅ LINE側で管理 |
| トークン保存 | localStorage | ⚠️ XSSリスクあり |

### セキュリティ上の問題点

#### 🟡 Medium

1. **LINE User ID の直接使用**
   ```
   問題: profiles.id = LINE User ID をそのまま使用
   リスク: LINE User ID が漏洩すると他人のデータにアクセス可能
   対策: RLSポリシーで自分のデータのみアクセス可能にする
   ```

2. **ANON_KEY のクライアント側露出**
   ```
   問題: フロントエンドコードに ANON_KEY が含まれる
   影響: 必然的だが、RLSでの防御が必須
   対策: 後述の「データアクセス制御」参照
   ```

## データアクセス制御

### 現在のRLS設定

#### profiles テーブル

**現在のポリシー:**
```sql
-- 全員が読み取り可能
CREATE POLICY "allow_public_read"
  ON profiles FOR SELECT
  USING (true);  -- ⚠️ 問題: すべてのユーザーが全プロフィールを閲覧可能

-- 全員が挿入可能
CREATE POLICY "allow_public_insert"
  ON profiles FOR INSERT
  WITH CHECK (true);  -- ⚠️ 問題: 他人のIDでレコード作成可能

-- 全員が更新可能
CREATE POLICY "allow_public_update"
  ON profiles FOR UPDATE
  USING (true);  -- ⚠️ 問題: 他人のプロフィールを更新可能
```

**問題点:**
- ❌ ユーザーAがユーザーBのプロフィールを読み取れる
- ❌ ユーザーAがユーザーBのスタンプ数を変更できる
- ❌ 他人のLINE User IDを使って不正なプロフィールを作成できる

**推奨ポリシー（本番環境）:**
```sql
-- 自分のプロフィールのみ閲覧可能
DROP POLICY "allow_public_read" ON profiles;
CREATE POLICY "user_read_own_profile"
  ON profiles FOR SELECT
  USING (
    id = current_setting('request.jwt.claims', true)::json->>'user_id'
    -- ⚠️ 注: Supabase Auth を使用していないため、別の方法が必要
  );

-- LIFFアプリの場合の代替案
-- アプリケーション層でフィルタリング
-- または、Supabase RPC 関数を使用
```

#### stamp_history テーブル

**現在のポリシー:**
```sql
-- 全員が読み取り可能
CREATE POLICY "allow_public_read"
  ON stamp_history FOR SELECT
  USING (true);  -- ⚠️ 問題: すべてのユーザーの履歴を閲覧可能

-- 全員が挿入可能

  ON stamp_history FOR INSERT
  WITH CHECK (true);  -- ⚠️ 問題: 他人のスタンプ履歴を作成可能
```

**推奨ポリシー:**
```sql
-- 自分のスタンプ履歴のみ閲覧可能
DROP POLICY "allow_public_read" ON stamp_history;

  -- ⚠️ 注: current_user_line_id() 関数を作成する必要あり

-- 自分のスタンプ履歴のみ挿入可能
DROP POLICY "allow_public_insert" ON stamp_history;
CREATE POLICY "user_insert_own_history"
  ON stamp_history FOR INSERT
```

### RLS制限の回避策（LIFFアプリ用）

Supabase Auth を使用していないため、以下の方法で対応：

#### 方法1: アプリケーション層でフィルタリング（現在）

```typescript
// ✅ 実装済み: フロントエンドでフィルタ
const { data: myProfile } = await supabase
  .from('profiles')
  .select('*')
  .eq('id', lineUserId)  // ← アプリ層でフィルタ
  .single();

// ⚠️ 問題: 悪意のあるユーザーがコードを改変すると他人のデータにアクセス可能
```

#### 方法2: RPC関数を使用（推奨）

```sql
-- Supabase に関数を作成
CREATE OR REPLACE FUNCTION get_my_profile(p_user_id TEXT)
RETURNS SETOF profiles
LANGUAGE sql
SECURITY DEFINER  -- SERVICE_ROLE_KEY の権限で実行
AS $$
  SELECT * FROM profiles WHERE id = p_user_id LIMIT 1;
$$;

-- RLS をバイパスしつつ、引数で制限
```

```typescript
// LIFFアプリから呼び出し
const { data, error } = await supabase
  .rpc('get_my_profile', { p_user_id: lineUserId });
```

## フロントエンドセキュリティ

### XSS対策

#### React の自動エスケープ

```tsx
// ✅ 安全（Reactが自動エスケープ）
<div>{profile.display_name}</div>
<div>{profile.real_name}</div>

// ❌ 危険（使用している箇所がないか確認が必要）
<div dangerouslySetInnerHTML={{ __html: userInput }} />
```

#### ユーザー入力の検証

```typescript
// ✅ 実装推奨: 入力値のサニタイズ
function sanitizeInput(input: string): string {
  return input
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#x27;')
    .slice(0, 200);  // 最大長制限
}
```

### データ検証

#### クライアント側検証

```typescript
// スタンプ数の範囲チェック
function validateStampCount(count: number): boolean {
  return Number.isInteger(count) && count >= 0 && count <= 999;
}

// 診察券番号のフォーマットチェック
function validateTicketNumber(ticket: string): boolean {
  return /^[A-Z0-9]{4,12}$/.test(ticket);
}
```

**⚠️ 注意:** クライアント側検証は改ざん可能なため、サーバー側（管理ダッシュボード側）でも必ず検証が必要

### localStorage のセキュリティ

#### LIFF SDK が保存するデータ

```
localStorage に保存される情報:
- liff.state.{liffId}: LIFF セッション情報
- liff.access_token: LINE アクセストークン
```

**セキュリティリスク:**
- ⚠️ XSS攻撃で localStorage の内容が盗まれる可能性
- ⚠️ 同じドメインの他のアプリケーションから

アクセス可能

**対策:**
1. ✅ HTTPS必須（中間者攻撃対策）
2. ✅ CSP (Content Security Policy) の設定
3. ⚠️ localStorage に機密情報を保存しない

## プライバシー保護

### 個人情報の取り扱い

#### LIFFアプリで取得する情報

| 情報 | 取得元 | 用途 | 保存場所 |
|-----|-------|------|---------|
| LINE User ID | liff.getProfile() | ユーザー識別 | profiles.id |
| 表示名 | liff.getProfile() | 画面表示 | profiles.display_name |
| プロフィール画像 | liff.getProfile() | 画面表示 | profiles.picture_url |
| 友だち登録状態 | liff.getFriendship() | マーケティング | profiles.is_line_friend |

#### 保存しない情報

- ❌ LINEアクセストークン（LIFF SDKが管理）
- ❌ 電話番号
- ❌ メールアドレス
- ❌ 住所

### ユーザー行動ログ

**event_logs テーブル:**
```typescript
// 記録される情報
await supabase.from('event_logs').insert({
  user_id: lineUserId,       // ⚠️ 個人識別子
  event_name: 'app_open',
  source: 'line_msg_260315',
  metadata: {
    user_agent: navigator.userAgent,  // ⚠️ デバイス情報
    screen_size: `${window.innerWidth}x${window.innerHeight}`
  }
});
```

**プライバシー対策:**
- ✅ 保存期間: 400日（約13ヶ月）で自動削除
- ⚠️ IPアドレスは記録していない
- ⚠️ 位置情報は記録していない

### LIFFアプリ側で実装すべきセキュリティ対策まとめ

```typescript
// security.ts - セキュリティユーティリティ

/**
 * 入力値のサニタイズ
 */
export function sanitizeInput(input: string, maxLength = 200): string {
  return input
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#x27;')
    .slice(0, maxLength);
}

/**
 * LINE User ID の検証
 */
export function isValidLineUserId(userId: string): boolean {
  return /^U[0-9a-f]{32}$/.test(userId);
}

/**
 * スタンプ数の検証
 */
export function validateStampCount(count: number): boolean {
  return Number.isInteger(count) && count >= 0 && count <= 999;
}

/**
 * 現在のユーザーIDを取得（LIFF SDK経由）
 */
export async function getCurrentUserId(): Promise<string> {
  if (!liff.isLoggedIn()) {
    throw new Error('ログインしてください');
  }

  const profile = await liff.getProfile();
  const userId = profile.userId;

  if (!isValidLineUserId(userId)) {
    throw new Error('無効なユーザーIDです');
  }

  return userId;
}

/**
 * 安全なSupabaseクエリ（自分のデータのみ）
 */
export async function safeQuery<T>(
  table: string,
  userId: string,
  selectFields = '*'
): Promise<T | null> {
  if (!isValidLineUserId(userId)) {
    throw new Error('無効なユーザーIDです');
  }

  const { data, error } = await supabase
    .from(table)
    .select(selectFields)
    .eq('id', userId)  // ← 必ず自分のIDでフィルタ
    .single();

  if (error) {
    console.error('Query error:', error.message);
    return null;
  }

  return data as T;
}
```
