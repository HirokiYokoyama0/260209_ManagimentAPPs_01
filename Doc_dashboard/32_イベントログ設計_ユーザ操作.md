# イベントログ設計（ユーザの操作ログ）

**作成日**: 2026-02-23
**目的**: ユーザー行動分析・マーケティング施策の効果測定
**方針**: Supabaseのみで完結するシンプルな設計

---

## 1. 背景・目的

### 実現したいこと
- **LINE配信の効果測定**: メッセージを送ってから何人がアプリを開いたか
- **未読率の把握**: 配信対象のうち、何%がアプリを開いていないか
- **ユーザー行動分析**: どの機能がよく使われているか、離脱ポイントはどこか
- **マーケティング施策の効果測定**: キャンペーン後の来院率変化など

### なぜSupabaseだけで良いのか
1. **リアルタイム性**: ログが溜まる様子を管理画面で即座に確認可能
2. **JSONB型の柔軟性**: 後から追加したいデータを自由に入れられる
3. **分析もSQLで完結**: 複雑な集計もSupabase上でSQLを叩くだけ
4. **コスト効率**: 外部サービス（Google Analytics等）の導入不要

### データ増加への対応
- パーティショニング: 月次テーブル分割で古いデータをアーカイブ
- TTL（Time To Live）: 一定期間後に自動削除するポリシー
- 集計テーブル: 日次で集計して元データは削除

```sql
create index idx_event_logs_created_at on event_logs(created_at desc);
create index idx_event_logs_source on event_logs(source) where source is not null;

-- RLSポリシー（セキュリティ）
alter table event_logs enable row level security;

-- ユーザーは自分のログのみ書き込み可能
create policy "Users can insert their own logs"
  on event_logs for insert
  with check (user_id = current_setting('app.current_user_id', true));

-- 管理者は全ログ参照可能（サービスロールキー使用）
create policy "Admins can view all logs"
  on event_logs for select
  using (true);

-- コメント追加（ドキュメント化）
comment on table event_logs is 'ユーザー行動ログ（マーケティング分析用）';
comment on column event_logs.event_name is 'イベント種別: app_open, stamp_scan, reward_exchange など';
comment on column event_logs.source is '流入元: line_msg_YYMMDD, direct, web など';
```

## 4. フロントエンド実装

### 4.1 共通ログ送信関数

```typescript
// lib/analytics.ts
import { supabase } from './supabase';

interface LogEventParams {
  eventName: string;
  source?: string;
  metadata?: Record<string, any>;
}

export const logEvent = async (params: LogEventParams) => {
  try {
    const { eventName, source, metadata } = params;

    // URLパラメータから流入元を取得
    const urlParams = new URLSearchParams(window.location.search);
    const sourceFromUrl = urlParams.get('from') || 'direct';

    // ユーザーエージェント等の基本情報を自動追加
    const enrichedMetadata = {
      ...metadata,
      user_agent: navigator.userAgent,
      screen_size: `${window.screen.width}x${window.screen.height}`,
      viewport_size: `${window.innerWidth}x${window.innerHeight}`,
      referrer: document.referrer || null,
      timestamp: new Date().toISOString(),
    };

    await supabase.from('event_logs').insert({
      event_name: eventName,
      source: source || sourceFromUrl,
      metadata: enrichedMetadata,
    });
  } catch (error) {
    // ログ送信失敗してもユーザー体験を妨げない
    console.error('⚠️ Analytics error:', error);
  }
};
```

```sql
with campaign_targets as (
  select unnest(target_user_ids) as user_id, sent_at
  from message_delivery_logs
  where campaign_id = 'msg_0222'
),
app_opens as (
  select distinct user_id
  from event_logs
  where event_name = 'app_open'

from campaign_targets ct
left join app_opens ao on ct.user_id = ao.user_id;
```

### 5.8 誰がいつスタンプ数を確認したか

```sql
-- スタンプページ閲覧履歴（直近7日）
select
  p.display_name,
  p.ticket_number,
  el.event_name,
  el.created_at as viewed_at,

-- スタンプ確認頻度ランキング（アクティブユーザー）
select
  p.display_name,
  count(*) as view_count,
  min(el.created_at) as first_view,
  max(el.created_at) as last_view,
  round(extract(epoch from (max(el.created_at) - min(el.created_at))) / 86400, 1) as days_between_first_and_last
from event_logs el
join profiles p on el.user_id = p.id
where el.event_name in ('stamp_page_view', 'home_page_view')
  and el.created_at >= current_date - interval '30 days'
group by p.id, p.display_name
having count(*) >= 3  -- 3回以上閲覧したユーザー
order by view_count desc;
```

### 5.9 ユーザーの行動フロー分析

```sql
-- 特定ユーザーの行動履歴（時系列）
select
  el.created_at,
  el.event_name,
  el.source,
  el.metadata
from event_logs el
where el.user_id = 'U5c70cd61...'  -- 特定ユーザーID

order by el.created_at asc;

-- 離脱ポイント分析（セッション終了前の最後のアクション）
with last_actions as (
  select
    user_id,
    event_name,
    created_at,
    lead(event_name) over (partition by user_id order by created_at) as next_event
  from event_logs
  where created_at >= current_date - interval '7 days'
)
select
  event_name as last_action_before_exit,
  count(*) as exit_count
from last_actions
where next_event is null  -- 次のアクションがない = セッション終了
  and event_name != 'session_end'
group by event_name
order by exit_count desc;
```

### 6.2 TTL（自動削除ポリシー）

```sql
-- 90日以上前のログを自動削除（毎日実行）
create or replace function delete_old_event_logs()
returns void as $$
begin
  delete from event_logs
  where created_at < current_date - interval '90 days';
end;
$$ language plpgsql;

-- Supabase Edge Functionsまたは外部cronで定期実行
```

### 6.3 集計テーブル（サマリー化）

```sql
-- 日次集計テーブル
create table event_logs_daily_summary (
  date date primary key,
  total_events int,
  unique_users int,
  event_counts jsonb,  -- {app_open: 1234, stamp_scan: 567, ...}
  created_at timestamptz default now()
);

-- 毎日深夜に前日分を集計
create or replace function summarize_daily_events()
returns void as $$
begin
  insert into event_logs_daily_summary (date, total_events, unique_users, event_counts)
  select
    date(created_at) as date,
    count(*) as total_events,
    count(distinct user_id) as unique_users,
    jsonb_object_agg(event_name, count) as event_counts
  from (
    select event_name, count(*) as count, created_at
    from event_logs
    where date(created_at) = current_date - interval '1 day'
    group by event_name, created_at
  ) sub
  group by date(created_at);
end;
$$ language plpgsql;
```

## 8. プライバシー・セキュリティ考慮事項

### 8.1 個人情報保護
- **user_id のみ記録**: 名前や診察券番号は event_logs に保存しない
- **分析時に JOIN**: 必要な時だけ profiles テーブルと結合
- **匿名化オプション**: 必要に応じて user_id をハッシュ化

### 8.2 データ保持期間
- **推奨**: 90日間（3ヶ月）で自動削除
- **長期保存が必要な場合**: 日次サマリーテーブルに集計して元データ削除

### 8.3 RLSポリシー
- ユーザーは自分のログのみ書き込み可能
- 閲覧は管理者（サービスロールキー）のみ

```
   ↓

   - 配信数: 100人
   - 開封数: 35人
   - 開封率: 35%
```

### 12.4 CSV出力

```typescript
const exportToCSV = (data: any[], filename: string) => {
  const headers = Object.keys(data[0]);
  const csvContent = [
    headers.join(','),
    ...data.map(row => headers.map(h => JSON.stringify(row[h])).join(','))
  ].join('\n');
  const bom = '\uFEFF';
  const blob = new Blob([bom + csvContent], { type: 'text/csv;charset=utf-8;' });
  const link = document.createElement('a');
  link.href = URL.createObjectURL(blob);
  link.download = filename;
  link.click();
};
```
