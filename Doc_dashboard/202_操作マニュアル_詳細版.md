## 1.2 システム概要

### つくばホワイト歯科 管理ダッシュボードとは

このシステムは、歯科医院の患者管理とLINEミニアプリ（スタンプカード）を統合した管理画面です。

### 主な機能

| 機能カテゴリ | 機能名 | 説明 |
|------------|-------|------|
| **患者管理** | 患者一覧 | 全患者の情報を一覧表示 |
| | 検索・フィルタ | 名前・診察券番号で検索 |
| | 情報編集 | 診察券番号・本名・来院日の編集 |
| **スタンプ** | スタンプ変更 | 来院時のスタンプ付与 |
| | 履歴管理 | スタンプ取得履歴の確認 |
| **メッセージ** | 個別送信 | 患者個別へのメッセージ |
| | 定型文 | よく使うメッセージのテンプレート |
| **ケア記録** | 治療記録 | 歯の治療状況の記録 |
| | 歯並び図 | ビジュアルで歯の状態を管理 |
| **分析** | スタンプ分布 | スタンプ数の統計 |
| | 来院傾向 | 来院頻度の分析 |
| | イベントログ | ユーザー行動の追跡 |

### システム構成

```
[管理ダッシュボード]        [LINEミニアプリ（患者用）]
        ↓                            ↓
    [Next.js API]              [Supabase RPC]
        ↓                            ↓
              [Supabase Database]
```

# 第2章: ログイン・ログアウト

## 2.1 初回ログイン

### 手順

1. ブラウザを開く（Google Chrome または Microsoft Edge 推奨）

2. 管理者から教わったURLにアクセス
   ```
   https://あなたのサイト.vercel.app/admin
   ```

3. ログイン画面が表示されます

4. 初期アカウント情報を入力
   - **ログインID**: `admin`
   - **パスワード**: `admin`

5. 「ログイン」ボタンをクリック

6. 患者一覧画面が表示されれば成功

### 💡 初回ログイン後の推奨作業

1. パスワードを変更する（[2.3 パスワード変更](#23-パスワード変更)参照）
2. ブラウザのブックマークに登録する
3. 各機能を試してみる

---

## 2.2 通常ログイン

### 手順

1. ブックマークまたはURLから管理画面にアクセス

2. **自動ログイン**
   - 前回ログインから7日以内の場合、自動的にログインされます
   - セッションが有効な場合、ログイン画面はスキップされます

3. **手動ログイン**
   - セッションが切れている場合、ログイン画面が表示されます
   - ログインIDとパスワードを入力してログイン

---

## 2.3 パスワード変更

⚠️ **重要**: 現在のバージョンでは、パスワード変更機能は管理者が環境変数を変更する必要があります。

### システム管理者向け手順

パスワードを変更するには、以下の環境変数を設定します：

```bash
# .env.local または Vercel の環境変数設定
ADMIN_USER=admin              # ログインID（変更可能）
ADMIN_PASSWORD=新しいパスワード  # パスワード（変更必須）
AUTH_SECRET=ランダムな文字列    # セッション署名用（32文字以上推奨）
```

変更後、アプリケーションを再起動してください。

---

## 2.4 ログアウト

### 手順

1. 画面右上の **「ログアウト」** ボタンをクリック

2. ログイン画面に戻ります

3. ブラウザを閉じる

### ⚠️ 注意事項

- **離席時は必ずログアウト**してください
- ログアウトしないと、他の人がアクセスできてしまいます
- セキュリティ保護のため、長時間操作しない場合も自動的にログアウトされます（タイムアウト: 約1時間）

---

## 2.5 セッション管理

### セッションの仕組み

- ログイン時に署名付きCookie（`admin_session`）が発行されます
- このCookieは7日間有効です
- ブラウザを閉じてもセッションは維持されます

### セッション切れの対処

セッションが切れると、以下のような動作になります：

1. 操作中に突然ログイン画面にリダイレクトされる
2. 「セッションが無効です」というメッセージが表示される

**対処方法:**
- 再度ログインしてください
- 作業中だった場合、再度操作をやり直してください

# 第4章: 患者管理

## 4.1 患者を探す

### 方法1: リアルタイム検索（推奨）

画面上部の検索ボックスを使用します。

#### 手順

1. **「患者を検索...」** と書かれた検索ボックスをクリック

2. 検索キーワードを入力
   - **名前**: 「山田」「太郎」など
   - **診察券番号**: 「12345」など

3. 入力と同時に、テーブルがリアルタイムで絞り込まれます

4. 検索結果が1件の場合、その患者だけが表示されます

5. 検索をクリアするには、検索ボックスの内容を削除します

#### 検索のコツ

- **部分一致**: 「山」と入力すると「山田」「小山」などが表示されます
- **スペース区切り**: 「山田 太」で「山田太郎」を検索できます
- **診察券番号**: 数字だけで検索すると正確です

### 方法2: テーブルをスクロール

患者数が少ない場合や、目視で探す場合に使用します。

#### 手順

1. テーブルを上下にスクロール

2. 該当する患者を見つける

#### 💡 ポイント

- テーブルは「最終来院日」の新しい順に並んでいます
- 患者数が多い場合は検索ボックスの使用を推奨します

---

## 4.2 患者情報の表示

### 表示される情報

テーブルに表示される各項目の詳細：

#### 名前

- **本名優先**: `real_name`（本名）が登録されている場合、本名を表示
- **LINE表示名**: 本名が未登録の場合、`display_name`（LINEの表示名）を表示

#### 診察券番号

- 登録済み: 番号を表示（例: `12345`）
- 未登録: `—`（ダッシュ）を表示

#### スタンプ数

- バッジ形式で表示（例: `5個`）
- 色分け:
  - 0個: グレー
  - 1-5個: ブルー
  - 6-10個: グリーン
  - 11個以上: パープル

#### 最終来院日

- 登録済み: 日付を表示（例: `2026/01/15`）
- 未登録: `—`（ダッシュ）を表示

#### 最新ログイン

- LINEミニアプリに最後にアクセスした日時
- 形式: `YYYY/MM/DD HH:mm`

#### 公式アカ友だち

- 友だち登録済み: `○ 済`（緑色）
- 未登録または不明: `—`（グレー）

💡 **友だち登録の重要性**: 友だち登録していない患者には、プッシュ通知が届きません。

---

## 4.3 患者情報の編集

### 編集可能な項目

- 診察券番号
- 本名
- 最終来院日

### 手順

1. 患者を検索して見つける

2. その患者の行にある **「編集」** ボタンをクリック

3. **プロフィール編集ダイアログ**が開きます

4. 以下の項目を編集できます：

   #### 診察券番号（Ticket Number）
   - テキスト入力
   - 例: `12345`、`A-001`など
   - 空欄にすることも可能

   #### 本名（Real Name）
   - テキスト入力
   - 例: `山田 太郎`
   - LINEの表示名と異なる場合に登録

   #### 最終来院日（Last Visit Date）
   - 日付入力
   - カレンダーから選択
   - 空欄にすることも可能

5. 「保存」ボタンをクリック

6. データが更新され、ダイアログが閉じます

7. テーブルの表示が即座に更新されます

### ⚠️ 注意事項

- **スタンプ数はここでは変更できません**（スタンプ数変更機能を使用）
- 本名は管理画面専用で、患者のアプリには表示されません
- 診察券番号は患者が予約時に使用するため、正確に入力してください

---

## 4.4 表示モードの切り替え

患者のLINEミニアプリの表示モードを切り替えます。

### 表示モードとは

- **大人モード（adult）**: 通常のデザイン
- **キッズモード（kids）**: 子ども向けのカラフルなデザイン

### 切り替え方法

1. 患者テーブルの **「表示モード」** 列にあるスイッチを確認

2. スイッチをクリック
   - 左（青）: 大人モード
   - 右（ピンク）: キッズモード

3. 即座にデータベースが更新されます

4. 患者が次回LINEミニアプリを開いたときに、新しいモードで表示されます

### 💡 使い分けの例

- **大人モード**: 一般の患者、保護者
- **キッズモード**: 子どもの患者

---

## 4.5 家族機能

複数の家族メンバーでスタンプを共有できる機能です。

### 家族とは

- 1つの家族 = 複数のメンバー
- 家族全体でスタンプを合算できます
- 代表者（parent）と子メンバー（child）の役割があります

### 家族の作成

1. 患者を検索して見つける

2. その患者の行にある **「...」メニュー** をクリック

3. **「家族作成」** を選択

4. ダイアログが開きます

5. 以下を設定：
   - **家族名**: 例: `山田家`、`田中さんの家族`
   - **代表者**: 選択した患者が自動的に代表者になります

6. 「作成」ボタンをクリック

7. 新しい家族が作成され、選択した患者がその家族のメンバーになります

### 家族へのメンバー追加

現在のバージョンでは、LINEミニアプリ側から家族メンバーを追加します。管理画面からの追加機能は今後実装予定です。

### 家族情報の確認

# 第5章: スタンプ管理

## 5.1 スタンプの仕組み

### スタンプとは

- 来院ごとに患者に付与されるポイント
- 基本: **1来院 = 1スタンプ**
- スタンプを貯めると特典と交換できます

### 内部の仕組み

💡 **技術詳細**: スタンプは内部的には「10倍の整数」で管理されています。

- 表示: `5個`
- データベース: `stamp_count = 50`（10倍）

これにより、将来的に小数点スタンプ（例: 0.5個）にも対応可能です。

### スタンプの自動更新

スタンプ数は以下のタイミングで自動更新されます：

1. **来院時にQRコードをスキャン**
   - `stamp_history` テーブルに履歴が追加
   - トリガーが発火
   - `profiles.stamp_count` が自動更新

2. **管理画面から手動変更**
   - スタンプ数変更機能を使用
   - 直接 `profiles.stamp_count` が更新

---

## 5.2 スタンプ数の変更

### 手順

1. 患者を検索して見つける

2. その患者の行にある **「...」メニュー** をクリック

3. **「スタンプ数変更」** を選択

4. **スタンプ数変更ダイアログ**が開きます

5. 以下が表示されます：
   - 現在のスタンプ数（例: `現在: 5個`）
   - 入力フィールド

6. 新しいスタンプ数を入力
   - 例: 来院した場合 → `6`（現在値 + 1）
   - 例: 修正する場合 → 正しい数値を入力

7. 「保存」ボタンをクリック

8. スタンプ数が更新され、テーブルの表示が即座に反映されます

### 💡 ポイント

- **増減どちらも可能**: 間違えた場合は減らすこともできます
- **0個にリセット**: `0`と入力すると0個にできます
- **範囲**: 0〜999個まで入力可能

### ⚠️ 注意事項

- スタンプ数を変更しても、`stamp_history`（履歴）には記録されません
- 履歴を残したい場合は、別途QRコードスキャン機能を使用してください

## 5.3 スタンプ履歴の確認

各患者のスタンプ取得履歴を確認できます。

### 手順

1. 患者を検索して見つける

2. その患者の行をクリックして詳細パネルを開く（将来実装予定）

または

1. データベースを直接確認（システム管理者のみ）

### 履歴に記録される情報

- 取得日時
- 取得方法（QRスキャン、手動追加など）
- 取得後のスタンプ数
- 備考

## 5.4 特典交換

患者がスタンプを使って特典と交換した履歴を管理します。

### 特典交換履歴の確認

1. サイドバーの **「🎁 特典交換履歴」** をクリック

2. 特典交換履歴画面が開きます

3. 以下の情報が表示されます：
   - 交換日時
   - 患者名
   - 特典名
   - 使用スタンプ数
   - ステータス（pending / completed / cancelled）

### ステータスの意味

| ステータス | 説明 | 対応 |
|-----------|------|------|
| **pending** | 交換申請済み | 受付で特典を提供する前 |
| **completed** | 提供完了 | 受付で実際に特典を渡した後 |
| **cancelled** | キャンセル | 誤交換などの取り消し |

### 特典の提供手順

1. 患者が LINEミニアプリで特典交換を申請

2. 管理ダッシュボードの「特典交換履歴」に `pending` として表示される

3. 受付で患者に特典を提供

4. ステータスを `completed` に変更（将来実装予定）

## 6.4 LINE連携について

### LINE Messaging APIとは

LINEの公式アカウントから患者に直接メッセージを送信できる機能です。

### 連携の仕組み

```
[管理ダッシュボード]
    ↓ メッセージ送信
[Next.js API]
    ↓ LINE Messaging API 呼び出し
[LINE サーバー]
    ↓ プッシュ通知
[患者のLINE]
```

### 必要な設定

以下の環境変数が設定されている場合、LINE連携が有効になります：

- `LINE_CHANNEL_ID`
- `LINE_CHANNEL_SECRET`
- または `LINE_CHANNEL_ACCESS_TOKEN`

### ⚠️ 注意事項

- **友だち登録必須**: 患者がLINE友だち登録していない場合、プッシュ通知は届きません
- **メッセージ料金**: LINE Messaging APIの使用には料金がかかる場合があります
- **送信制限**: LINEのレート制限に注意してください

# 第7章: ケア記録機能

## 7.1 ケア記録とは

患者の歯の治療状況をビジュアルで記録・管理する機能です。

### 主な機能

- 32本の永久歯をビジュアル表示
- 各歯の治療状況を色分け
- 治療履歴の記録
- 患者アプリへの自動反映

### 記録できる情報

- 歯の状態（虫歯治療済、経過観察など）
- スタッフ専用メモ
- 次回来院メモ（患者にも表示）

---

## 7.2 歯並び図の見方

### 歯の番号付け（ISO 3950方式）

永久歯は以下のように番号付けされます：

```
        上顎
右                       左
18 17 16 15 14 13 12 11 | 21 22 23 24 25 26 27 28
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
48 47 46 45 44 43 42 41 | 31 32 33 34 35 36 37 38
        下顎
```

**覚え方:**
- 第1数字: 象限（1=右上, 2=左上, 3=左下, 4=右下）
- 第2数字: 位置（1=中切歯, 8=第三大臼歯）

### 歯の色分け

| 色 | 状態 | 意味 |
|----|------|------|
| 🟢 緑 | 虫歯治療済 | 治療完了、問題なし |
| 🟡 黄 | 経過観察 | 要注意、経過を見る |
| 🔴 赤 | 治療予定 | 治療が必要 |
| 🟠 オレンジ | 治療中 | 現在治療中 |
| 🔵 青 | 被せ物 | クラウン・インレー装着済み |
| 🔷 水色 | 歯石除去 | スケーリング完了 |
| 🟣 紫 | クリーニング | 予防クリーニング |
| ⚪ グレー | 記録なし | 状態未記録 |

---

## 7.3 記録の入力方法

### 手順

1. 患者を検索して見つける

2. その患者の行にある **「🦷 ケア記録」** ボタンをクリック

3. ケア記録画面が開きます

4. **歯並び図**が表示されます
   - 上顎: 18〜28番
   - 下顎: 48〜38番

5. **記録したい歯をクリック**
   - 歯がハイライト表示されます
   - 選択された歯番号が表示されます

6. **状態ボタンをクリック**して状態を選択
   - 虫歯治療済（緑）
   - 経過観察（黄）
   - 治療予定（赤）
   - 治療中（オレンジ）
   - 被せ物（青）
   - 歯石除去（水色）
   - クリーニング（紫）
   - クリア（記録を削除）

7. 歯の色が変わったら選択成功です

8. **メモを入力**（任意）
   - **スタッフ専用メモ**: 患者には見えません（最大500文字）
   - **次回来院メモ**: 患者のアプリにも表示されます（最大200文字）

9. 複数の歯を記録する場合は、手順5〜7を繰り返します

10. 「保存」ボタンをクリック

11. 記録が保存され、確認ダイアログが表示されます

12. 患者のLINEミニアプリにも即座に反映されます

### 💡 ポイント

- **複数の歯を一度に記録可能**: 保存ボタンを押す前に、複数の歯を選択・変更できます
- **上書き保存**: 既に記録がある歯を変更すると、新しい状態で上書きされます
- **記録の削除**: 「クリア」ボタンで記録を削除できます

過去のケア記録履歴を確認できます。

3. 最新5件の履歴が表示されます：

## 7.5 メモの使い分け

2種類のメモがあり、用途が異なります。

### スタッフ専用メモ

**用途:**

**例:**

**表示範囲:**

### 💡 使い分けの基準

| 内容 | スタッフ専用メモ | 次回来院メモ |
|-----|----------------|-------------|
| 治療の詳細（専門用語） | ✅ | ❌ |
| 患者への注意事項 | ❌ | ✅ |
| 次回の治療内容（わかりやすく） | ❌ | ✅ |
| 内部的な申し送り | ✅ | ❌ |
| 持ち物のお願い | ❌ | ✅ |

# 第10章: トラブルシューティング

## 10.1 よくある問題と解決方法

### 問題1: ログインできない

**症状:** ログイン画面でID・パスワードを入力しても「認証に失敗しました」と表示される

**原因と対処:**

1. **ID・パスワードの入力ミス**
   - 大文字・小文字を確認
   - スペースが入っていないか確認
   - コピー&ペーストを試す

2. **パスワードが変更されている**
   - 管理者にパスワードを確認

3. **ブラウザのCookieが無効**
   - ブラウザの設定でCookieを有効にする

---

### 問題2: 患者が見つからない

**症状:** 検索しても患者が表示されない

**原因と対処:**

1. **検索キーワードの間違い**
   - 全角・半角を確認
   - スペースを削除してみる

2. **患者が未登録**
   - 新規患者の場合、LINEミニアプリに初回アクセスすると自動登録されます
   - 登録されるまで待つ

3. **データ読み込みエラー**
   - ページをリロード（F5キー）

---

### 問題3: スタンプが変更できない

**症状:** スタンプ数変更ダイアログで保存しても反映されない

**原因と対処:**

1. **インターネット接続の問題**
   - インターネット接続を確認
   - Wi-Fiを再接続

2. **データベース接続エラー**
   - ページをリロード
   - 再度操作を試す

3. **ブラウザのキャッシュ**
   - ブラウザのキャッシュをクリア
   - Ctrl + Shift + Delete（Windows）
   - Cmd + Shift + Delete（Mac）

---

### 問題4: メッセージが送信できない

**症状:** メッセージ送信ボタンをクリックしてもエラーが表示される

**原因と対処:**

1. **メッセージ本文が空**
   - メッセージ本文を入力してください

2. **患者がLINE友だち登録していない**
   - メッセージはデータベースに保存されますが、プッシュ通知は届きません
   - 患者に友だち登録を依頼してください

3. **LINE API の設定エラー**
   - システム管理者に連絡してください

---

### 問題5: ケア記録が保存できない

**症状:** ケア記録画面で保存ボタンをクリックしてもエラーが表示される

**原因と対処:**

1. **データベース接続エラー**
   - インターネット接続を確認
   - ページをリロード

2. **メモの文字数オーバー**
   - スタッフ専用メモ: 500文字以内
   - 次回来院メモ: 200文字以内
   - 文字数を減らしてください

## 10.3 データの整合性確認

スタンプ数やデータに不整合がある場合の確認方法です（システム管理者向け）。

### Supabaseでの確認

1. Supabase ダッシュボードにアクセス

2. SQL エディタを開く

3. 以下のクエリを実行：

```sql
-- スタンプ数の整合性チェック
SELECT
  p.id AS user_id,
  p.stamp_count AS profile_stamp_count,
  COALESCE(MAX(sh.stamp_number), 0) AS history_max_stamp_number,
  CASE
    WHEN p.stamp_count = COALESCE(MAX(sh.stamp_number), 0) THEN '✅ 一致'
    ELSE '❌ 不一致'
  END AS status
FROM profiles p
LEFT JOIN stamp_history sh ON p.id = sh.user_id
GROUP BY p.id, p.stamp_count
HAVING p.stamp_count != COALESCE(MAX(sh.stamp_number), 0);
```

不整合がある場合、管理者に連絡してください。

# 第11章: セキュリティとベストプラクティス

## 11.1 セキュリティガイドライン

### 基本原則

1. **パスワード管理**
   - 初期パスワード（`admin`）は必ず変更する
   - パスワードは定期的に変更する（3ヶ月ごと推奨）
   - パスワードをメモに書かない
   - 他人に教えない

2. **ログアウトの徹底**
   - 離席時は必ずログアウト
   - 業務終了後はログアウト
   - ブラウザを閉じるだけでなく、ログアウトボタンをクリック

3. **画面ロック**
   - PCを離れる際は画面をロック
   - Windows: Windowsキー + L
   - Mac: Control + Command + Q

---

## 11.2 個人情報の取り扱い

### 管理画面で扱う個人情報

- 患者の名前
- 診察券番号
- LINE User ID
- 来院履歴
- 治療記録

### 取り扱いの注意事項

1. **画面の覗き見防止**
   - 患者から見えない場所で操作する
   - モニターにプライバシーフィルターを貼る

2. **画面のスクリーンショット禁止**
   - 患者情報の画面をスクリーンショットしない
   - SNSへの投稿は厳禁

3. **印刷の禁止**
   - 患者情報を印刷しない
   - 必要な場合は管理者に相談

4. **第三者への情報提供禁止**
   - 患者情報を院外に持ち出さない
   - メールやチャットで送信しない

## 11.3 操作ログについて

### 記録される操作

管理画面での操作は `activity_logs` テーブルに記録されます：

- ログイン
- 患者情報の編集
- スタンプ数の変更
- メッセージの送信
- ケア記録の入力

### ログの用途

- セキュリティ監査
- 不正操作の検知
- トラブル発生時の原因調査

### ⚠️ 重要

すべての操作は記録されています。不正な操作や個人情報の不適切な取り扱いは検知されます。
