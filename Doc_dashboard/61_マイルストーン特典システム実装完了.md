## ✅ 実装内容

### 1. マイルストーン判定ロジック

**ファイル:** [lib/milestones.ts](lib/milestones.ts)

**実装した機能:**

#### 1-1. マイルストーン判定

```typescript
export function checkMilestones(
  oldStampCount: number,
  newStampCount: number
): Array<{ milestone: number; rewardType: RewardType }>
```

- スタンプ数変更時に到達したマイルストーンを判定
- 10の倍数、50の倍数、300+150の倍数を検出
- 優先度ルールを適用（自費メニュー > POIC > 歯ブラシ）

**例:**
- 95 → 105: マイルストーン 100 (POIC)
- 290 → 310: マイルストーン 300 (自費メニュー), 310 (歯ブラシ)

#### 1-2. 特典自動付与

```typescript
export async function grantMilestoneReward(
  userId: string,
  milestone: number,
  rewardType: RewardType
)
```

- `milestone_rewards` テーブルから特典を取得
- POIC初回判定（初回は本体込、2回目以降は補充用）
- 有効期限の計算:
  - 歯ブラシ: 当日限り（23:59:59まで）
  - POIC: 5ヶ月間
  - 自費メニュー: 5ヶ月間
- `reward_exchanges` に `is_milestone_based = true` でレコード作成
- `milestone_history` に到達履歴を記録

#### 1-3. スタンプ付与時の統合処理

```typescript
export async function handleStampMilestones(
  userId: string,
  oldStampCount: number,
  newStampCount: number
)
```

- マイルストーン判定 → 特典付与の一連の処理
- エラーハンドリング（特典付与失敗でもスタンプ付与は成功）

## 🎯 動作仕様

### マイルストーンと特典の対応

| 累積スタンプ数 | 特典内容 | 有効期限 | 優先度 |
|--------------|---------|----------|--------|
| **10, 20, 30, 40...** | 歯ブラシ 1本 | 当日限り | 1（低） |
| **50, 100, 150, 200...** | POIC殺菌剤 | 5ヶ月間 | 2（中） |
| **300, 450, 600, 750...** | 選べる自費メニュー | 5ヶ月間 | 3（高） |

### 優先度ルールの例

**50スタンプ到達時:**
- 該当マイルストーン: 10の倍数（歯ブラシ）、50の倍数（POIC）
- **付与される特典:** POIC殺菌剤のみ
- **付与されない:** 歯ブラシ（優先度が低いためスキップ）

**100スタンプ到達時:**
- 該当マイルストーン: 10の倍数（歯ブラシ）、50の倍数（POIC）
- **付与される特典:** POIC殺菌剤（補充用）のみ
- **付与されない:** 歯ブラシ（優先度が低いためスキップ）

**300スタンプ到達時:**
- 該当マイルストーン: 10の倍数（歯ブラシ）、50の倍数（POIC）、300（自費メニュー）
- **付与される特典:** 選べる自費メニューのみ
- **付与されない:** 歯ブラシ、POIC（優先度が低いためスキップ）

### POIC初回/2回目の判定

- **初回:** `is_first_time = true`, 表示「初回のみ本体ごとプレゼント」
- **2回目以降:** `is_first_time = false`, 表示「2回目以降は補充用のみ」

### 有効期限の計算

```typescript
function calculateValidUntil(rewardType: RewardType): Date | null {
  if (rewardType === RewardType.TOOTHBRUSH) {
    // 歯ブラシ: 当日限り（翌日0時まで）
    const validUntil = new Date();
    validUntil.setHours(23, 59, 59, 999);
    return validUntil;
  }

  if (rewardType === RewardType.POIC || rewardType === RewardType.PREMIUM_MENU) {
    // POIC・自費メニュー: 5ヶ月間
    const validUntil = new Date();
    validUntil.setMonth(validUntil.getMonth() + 5);
    return validUntil;
  }

  return null;
}
```

## 📊 データベースの状態

### テーブル構成

| テーブル名 | 状態 | 用途 |
|-----------|------|------|
| `rewards` | ✅ 維持 | 旧データの参照用（4件） |
| `milestone_rewards` | ✅ 作成済み | 新データ用（3件） |
| `reward_exchanges` | ✅ カラム追加 | 新旧両方のデータを格納 |
| `milestone_history` | ✅ 作成済み | マイルストーン到達履歴 |

### milestone_rewards の内容

| name | reward_type | milestone_type | validity_months |
|------|-------------|----------------|----------------|
| 歯ブラシ 1本 | toothbrush | every_10 | 0（当日限り） |
| POIC殺菌剤 | poic | every_50 | 5（5ヶ月） |
| 選べる自費メニュー | premium_menu | every_150_from_300 | 5（5ヶ月） |

### reward_exchanges の新カラム

| カラム名 | 型 | 説明 |
|---------|---|------|
| `milestone_reached` | INTEGER | 到達したマイルストーン数（10, 50, 300...） |
| `valid_until` | TIMESTAMPTZ | 有効期限 |
| `is_first_time` | BOOLEAN | 初回特典かどうか（POIC用） |
| `is_milestone_based` | BOOLEAN | マイルストーン型特典か（true = 新, false = 旧） |

### 新旧データの区別

```sql
-- 旧データ（手動交換型）
SELECT * FROM reward_exchanges
WHERE is_milestone_based = false OR is_milestone_based IS NULL;
-- → 既存の7件 → rewards テーブルを参照

-- 新データ（マイルストーン型）
SELECT * FROM reward_exchanges
WHERE is_milestone_based = true;
-- → 今後作成されるレコード → milestone_rewards テーブルを参照
```

## ⚠️ 重要な注意事項

### 1. 既存データの保護

**✅ 確認済み:**
- `rewards` テーブルは削除されていない（4件のまま）
- 既存の7件の交換履歴は `is_milestone_based = false` または `NULL` で保護されている
- 外部キー制約は維持されている

### 2. エラーハンドリング

**実装済みの安全策:**
- マイルストーン処理でエラーが発生しても、スタンプ付与は成功する
- 特典付与エラーは個別に処理され、他のマイルストーンには影響しない
- エラーログは詳細に出力される

### 3. パフォーマンス

**考慮事項:**
- マイルストーン判定は軽量（計算のみ）
- データベースクエリは最小限（特典取得、POIC初回判定、レコード作成）
- 大量のスタンプ付与（例: 0 → 1000）でも処理可能
