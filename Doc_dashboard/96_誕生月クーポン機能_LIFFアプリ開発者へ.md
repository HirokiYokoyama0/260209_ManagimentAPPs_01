## 📋 概要

誕生月に特別クーポンを配信する機能を実装します。つきましては、LIFFアプリ側での対応をお願いいたします。

---

## 🎯 実装目的

- **誕生月クーポン配信**: 患者の誕生月に、LINE公式アカウントからFlex Message（カードタイプメッセージ）で誕生日クーポンを送信
- **患者エンゲージメント向上**: 誕生月という特別なタイミングでコミュニケーションを強化
- **来院促進**: 10%OFFクーポンによる誕生月の来院促進

---

## 🗄️ データベース変更

### 1. profilesテーブルへの追加カラム

```sql
-- 新規カラム: birth_month
ALTER TABLE profiles
  ADD COLUMN birth_month INTEGER DEFAULT NULL
  CHECK (birth_month >= 1 AND birth_month <= 12);
```

**カラム仕様**:
- **カラム名**: `birth_month`
- **データ型**: `INTEGER`
- **制約**: `1-12`（1=1月、2=2月、...、12=12月）
- **NULL許容**: ✅ YES（既存ユーザーは未登録状態）
- **用途**: 誕生月クーポン配信の絞り込み

**プライバシー配慮**:
- 誕生日（生年月日）ではなく、**誕生月のみ**を管理
- 年齢や生年月日は保存しない

### 3. バリデーション

#### 必須チェック:
- 誕生月は**任意項目**（NULLを許容）
- ただし、入力された場合は`1-12`の範囲であることを確認

#### 実装例:

```typescript
function validateBirthMonth(value: number | null): boolean {
  if (value === null || value === undefined) {
    return true;  // NULL許容
  }
  return value >= 1 && value <= 12;
}
```

### 4. 既存ユーザーの対応

#### パターンA: 既存ユーザーには後から入力を促す
- プロフィール編集画面で誕生月を追加
- 「誕生月を登録すると、お誕生日クーポンが届きます！」などのメッセージで入力を促す

#### パターンB: 既存ユーザーにはポップアップで入力を促す
- アプリ起動時に `birth_month === NULL` のユーザーに対して、誕生月入力ダイアログを表示

### 1. 配信タイミング
- **毎月1日**: 管理ダッシュボードから、その月が誕生月のユーザーに配信

## ❓ Q&A

### Q1: 誕生月は必須項目ですか？
**A**: いいえ、**任意項目**です。NULLを許容します。ただし、誕生月を登録していないユーザーには誕生月クーポンは配信されません。

### Q2: 誕生日（日付）まで必要ですか？
**A**: いいえ、**誕生月のみ**で十分です。プライバシー配慮のため、生年月日は保存しません。

### Q3: 既存ユーザーはどうなりますか？
**A**: `birth_month = NULL` のまま残ります。LIFFアプリ側でプロフィール編集機能を通じて、後から入力してもらうことを想定しています。

### Q4: 誕生月を変更できますか？
**A**: はい、プロフィール編集画面で変更可能にしてください。誤入力の修正や、意図的な変更に対応するためです。

### Q5: 複数月にまたがる配信は可能ですか？
**A**: はい、管理ダッシュボードでは複数の誕生月を同時に選択できます（例: 1月、2月、3月を同時選択）。
